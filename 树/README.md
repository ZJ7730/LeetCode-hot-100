## 2. 本专题核心思想

- 二叉树题先判断使用 DFS 还是 BFS：需要按层输出通常用 BFS，其余多数用递归 DFS。
- DFS 的核心是明确当前递归函数的返回值含义：返回高度、返回节点、返回布尔值或直接修改树结构。
- BST 题优先想到中序遍历，因为 BST 中序结果严格递增。
- 树形 DP 常见写法是递归返回一个子树信息，同时用全局变量更新答案。
- 修改树结构时要注意递归顺序，例如展开为链表需要从右到左处理。
- LeetCode 面试通常已提供 `TreeNode`，当前目录既有公共 `TreeNode.java`，部分题也在类内重复定义节点结构。

## 3. 通用解题模板

### TreeNode 结构

```java
public class TreeNode {
    int val;
    TreeNode left;
    TreeNode right;
}
```

### DFS 递归模板

```java
void dfs(TreeNode root) {
    if (root == null) {
        return;
    }

    // 前序位置：处理 root
    dfs(root.left);
    // 中序位置：处理 root
    dfs(root.right);
    // 后序位置：处理 root
}
```

### 中序遍历模板

对应 `H_94.java`：

```java
void dfs(TreeNode root) {
    if (root == null) {
        return;
    }
    dfs(root.left);
    result.add(root.val);
    dfs(root.right);
}
```

### BFS 层序遍历模板

对应 `H_102.java`：

```java
Queue<TreeNode> queue = new LinkedList<>();
if (root != null) {
    queue.add(root);
}
while (!queue.isEmpty()) {
    int size = queue.size();
    List<Integer> res = new ArrayList<>();
    while (size-- > 0) {
        TreeNode node = queue.poll();
        res.add(node.val);
        if (node.left != null) {
            queue.add(node.left);
        }
        if (node.right != null) {
            queue.add(node.right);
        }
    }
    result.add(res);
}
```

### BST 中序校验模板

对应 `H_98.java`：

```java
TreeNode pre = null;

boolean isValidBST(TreeNode root) {
    if (root == null) {
        return true;
    }
    boolean left = isValidBST(root.left);
    if (pre != null && pre.val >= root.val) {
        return false;
    }
    pre = root;
    boolean right = isValidBST(root.right);
    return left && right;
}
```

### 树形 DP 模板

对应 `H_543.java`：

```java
int answer = 0;

int dfs(TreeNode root) {
    if (root == null) {
        return 0;
    }
    int left = dfs(root.left);
    int right = dfs(root.right);
    answer = Math.max(answer, left + right);
    return Math.max(left, right) + 1;
}
```

### 最近公共祖先模板

对应 `H_236.java`：

```java
TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null) {
        return null;
    }
    if (root == p || root == q) {
        return root;
    }
    TreeNode leftNode = lowestCommonAncestor(root.left, p, q);
    TreeNode rightNode = lowestCommonAncestor(root.right, p, q);
    if (leftNode != null && rightNode != null) {
        return root;
    }
    return leftNode != null ? leftNode : rightNode;
}
```

## 5. 每题详解

### H_94 - 二叉树的中序遍历

- 题目定位：返回二叉树的中序遍历结果。
- 当前代码思路：全局 `result`，递归顺序是左、根、右。
- 关键变量：`result` 保存遍历结果；`root == null` 是递归出口。
- 解题步骤：递归左子树；加入当前节点值；递归右子树。
- 复杂度：时间 `O(n)`，空间 `O(h)`。
- 面试易错点：中序是左根右，不要和前序、后序混淆。
- 可复述话术：中序遍历的递归定义就是先遍历左子树，再访问当前节点，最后遍历右子树。

### H_102 - 二叉树的层序遍历

- 题目定位：按层返回二叉树节点值。
- 当前代码思路：用队列保存当前待处理节点，每轮先记录当前层的 `size`。
- 关键变量：`size` 固定当前层节点数；`res` 保存当前层结果。
- 解题步骤：根节点入队；队列非空时处理一层；弹出节点并加入左右孩子；层结果加入 `result`。
- 复杂度：时间 `O(n)`，空间 `O(w)`。
- 面试易错点：必须先保存当前层 `size`，否则下一层节点会混入当前层。
- 可复述话术：BFS 按层遍历时，队列里先放根，每次处理队列当前长度个节点，这些节点就是同一层。

### H_104 - 二叉树的最大深度

- 题目定位：求根节点到最深叶子节点的节点数。
- 当前代码思路：递归返回当前子树最大高度。
- 关键变量：`leftHeight/rightHeight` 分别是左右子树高度。
- 解题步骤：空节点返回 0；递归求左右高度；返回较大值加 1。
- 复杂度：时间 `O(n)`，空间 `O(h)`。
- 面试易错点：深度按节点数计算，空树深度为 0。
- 可复述话术：当前树的最大深度等于左右子树最大深度中的较大者再加当前节点这一层。

### H_111 - 二叉树的最小深度

- 题目定位：求根到最近叶子节点的节点数。
- 当前代码思路：递归求左右深度，但单边为空时不能取 0 作为最小值。
- 关键变量：如果左子树为空右子树不空，返回 `1 + right`；反之返回 `1 + left`。
- 解题步骤：空节点返回 0；递归左右；处理单子树为空；双子树都存在时取最小值。
- 复杂度：时间 `O(n)`，空间 `O(h)`。
- 面试易错点：最小深度必须到叶子节点，不能把空子树当成深度 0 直接参与 `min`。
- 可复述话术：如果一个节点只有一边子树，最近叶子只能在非空那边，所以要单独处理。

### H_543 - 二叉树的直径

- 题目定位：求任意两个节点之间最长路径的边数。
- 当前代码思路：递归返回高度，同时用全局 `maxDiameter` 更新经过当前节点的最大路径。
- 关键变量：经过当前节点的直径是 `left + right`。
- 解题步骤：后序遍历；得到左右高度；更新全局直径；返回当前子树高度。
- 复杂度：时间 `O(n)`，空间 `O(h)`。
- 面试易错点：递归返回的是高度，最终答案更新的是直径，两者不是同一个含义。
- 可复述话术：每个节点都可能是最长路径的拐点，左右子树高度之和就是经过它的最长路径。

### H_226 - 反转二叉树

- 题目定位：交换每个节点的左右子树。
- 当前代码思路：当前节点先交换左右孩子，再递归处理交换后的左右子树。
- 关键变量：`swapChildren` 用 `tmp` 临时保存左子树。
- 解题步骤：空节点返回；交换左右；递归左右；返回根节点。
- 复杂度：时间 `O(n)`，空间 `O(h)`。
- 面试易错点：交换后继续递归左右都可以，因为每个节点只需交换一次。
- 可复述话术：反转整棵树可以拆成反转当前节点的左右孩子，再让左右子树各自完成同样操作。

### H_617 - 合并二叉树

- 题目定位：把两棵二叉树按相同位置合并。
- 当前代码思路：如果某棵树为空直接返回另一棵；都不空则新建节点，值为两节点值之和。
- 关键变量：`root.left/right` 分别递归合并左右子树。
- 解题步骤：处理空节点；创建新根；递归合并左子树和右子树；返回新根。
- 复杂度：时间 `O(n)`，空间 `O(h)`。
- 面试易错点：一边为空时可以直接复用另一边子树，不需要继续拆。
- 可复述话术：两棵树同步递归，当前位置都存在就相加，否则返回非空节点。

### H_114 - 二叉树展开为链表

- 题目定位：把二叉树按前序遍历顺序展开成右指针链表。
- 当前代码思路：使用反向前序遍历，也就是右、左、根；全局 `prev` 指向已经展开好的链表头。
- 关键变量：`root.right = prev`，`root.left = null`，再更新 `prev = root`。
- 解题步骤：先处理右子树；再处理左子树；当前节点接到 `prev` 前面。
- 复杂度：时间 `O(n)`，空间 `O(h)`。
- 面试易错点：如果按普通前序直接改指针，可能丢失原右子树；反向处理更安全。
- 可复述话术：目标是前序链表，所以从前序的反方向处理，每次把当前节点接到已处理链表前面。

### H_98 - 验证二叉搜索树

- 题目定位：判断二叉树是否满足 BST。
- 当前代码思路：中序遍历时用 `pre` 记录上一个访问节点，要求当前值严格大于 `pre.val`。
- 关键变量：`pre` 是中序遍历中的前驱节点。
- 解题步骤：递归左子树；比较 `pre` 和当前节点；更新 `pre`；递归右子树。
- 复杂度：时间 `O(n)`，空间 `O(h)`。
- 面试易错点：不能只比较当前节点和左右孩子，必须保证整个左子树都小、整个右子树都大。
- 可复述话术：BST 的中序遍历结果应该是严格递增的，所以只要中序过程中发现非递增就返回 false。

### H_538 - 把二叉搜索树转换为累加树

- 题目定位：把每个节点值改成原树中大于等于它的所有节点值之和。
- 当前代码思路：BST 中序是升序，反中序右、根、左就是降序，用 `prev` 累加已访问的大值。
- 关键变量：`prev` 保存当前节点右侧所有更大节点的累加和。
- 解题步骤：递归右子树；当前值加上 `prev`；更新 `prev`；递归左子树。
- 复杂度：时间 `O(n)`，空间 `O(h)`。
- 面试易错点：遍历方向必须是右根左，不是普通中序。
- 可复述话术：从最大值开始倒着遍历 BST，就能不断累加已经见过的更大值。

### H_101 - 对称二叉树

- 题目定位：判断二叉树是否关于中心轴对称。
- 当前代码思路：递归同时比较左子树和右子树的镜像位置。
- 关键变量：外侧是 `left.left` 和 `right.right`，内侧是 `left.right` 和 `right.left`。
- 解题步骤：两个节点都空返回 true；一个为空或值不同返回 false；递归比较外侧和内侧。
- 复杂度：时间 `O(n)`，空间 `O(h)`。
- 面试易错点：不是分别判断左右子树是否相同，而是交叉比较镜像位置。
- 可复述话术：对称要求左子树的左边等于右子树的右边，左子树的右边等于右子树的左边。

### H_236 - 二叉树的最近公共祖先

- 题目定位：找普通二叉树中两个节点的最近公共祖先。
- 当前代码思路：如果当前节点是 `p/q` 直接返回；递归左右子树；左右都有返回当前节点。
- 关键变量：`leftNode/rightNode` 表示左右子树中是否找到了目标节点或祖先。
- 解题步骤：空节点返回 null；命中目标返回 root；递归左右；根据左右返回值判断。
- 复杂度：时间 `O(n)`，空间 `O(h)`。
- 面试易错点：这是普通二叉树，不要使用 BST 大小关系。
- 可复述话术：如果左右子树各找到一个目标，当前节点就是最近公共祖先；如果只在一边找到，就把那边结果向上返回。

## 6. 面试高频易错点

- 最小深度不能直接 `1 + Math.min(left, right)`，要排除空子树。
- BST 验证要用中序递增或上下界，不能只看当前节点的直接孩子。
- 层序遍历每层开始前要固定 `size`。
- 二叉树直径的答案是边数，当前代码用 `left + right` 正好表示边数。
- 最近公共祖先题要先判断 `root == p || root == q`。
- 展开为链表时先处理右子树再处理左子树，可以避免原指针丢失。
