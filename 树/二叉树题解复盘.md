## H_98 - 验证二叉搜索树

### 核心思路

BST 的中序遍历结果必须严格递增：

```text
左 -> 根 -> 右
```

使用成员变量 `pre` 保存刚刚访问过的前一个中序节点。若当前节点不大于 `pre`：

```java
pre.val >= root.val
```

说明中序序列不再严格递增，当前树不是 BST。

### 面试速记

```text
BST 中序遍历必须严格递增：左 -> 根 -> 右，用 pre 记前一个中序节点，出现 pre.val >= root.val 就返回 false。
```

### 题解代码

```java
TreeNode pre = null;

public boolean isValidBST(TreeNode root) {
    pre = null;
    return dfs(root);
}

private boolean dfs(TreeNode root) {
    if (root == null) {
        return true;
    }

    if (!dfs(root.left)) {
        return false;
    }

    if (pre != null && pre.val >= root.val) {
        return false;
    }

    pre = root;
    return dfs(root.right);
}
```

### `pre` 的含义

`pre` 不是当前节点的父节点，而是中序遍历顺序中刚刚访问过的节点。

例如：

```text
      5
     / \
    1   7
       / \
      6   8
```

中序访问顺序：

```text
1 -> 5 -> 6 -> 7 -> 8
```

| 当前节点 | 进入时 `pre` | 比较结果 | 更新后 `pre` |
| --- | --- | --- | --- |
| `1` | `null` | 无需比较 | `1` |
| `5` | `1` | `1 < 5` | `5` |
| `6` | `5` | `5 < 6` | `6` |
| `7` | `6` | `6 < 7` | `7` |
| `8` | `7` | `7 < 8` | `8` |

全程严格递增，因此是合法 BST。

### 为什么不能只比较父子节点

反例：

```text
      5
     / \
    1   4
       / \
      3   6
```

右子树节点必须大于根节点 `5`，节点 `4` 不满足这一全局要求；但若只看局部父子关系：

```text
3 < 4 < 6
```

仍可能遗漏问题。中序序列为：

```text
1 -> 5 -> 3 -> 4 -> 6
```

访问到 `3` 时：

```text
pre = 5
5 >= 3
```

立刻判定不是 BST。

### 易错点

- 比较必须是 `>=`，因为 BST 不允许重复值。
- 比较节点引用或数值时，本题比较的是 `pre.val` 与 `root.val` 的大小，不是节点引用。
- 左子树已经不合法时应立即返回，避免无意义遍历；代码中的 `if (!dfs(root.left)) return false` 就是短路。
- `pre` 是成员变量，入口处重置为 `null`，避免多次调用时保留上一次树的状态。

### 复杂度

- 时间复杂度：`O(n)`，每个节点最多访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。

## H_102 - 二叉树的层序遍历

### 核心思路

使用队列进行 BFS。队列保存待访问节点，每次外层循环处理一整层节点，并将这一层的节点值收集到一个列表中。

关键是每层开始时保存：

```java
int size = queue.size();
```

`size` 表示当前层节点数量。即使处理当前层时将孩子节点加入队列，也只会处理最开始的 `size` 个节点，孩子会留给下一层。

### 面试速记

```text
BFS 层序遍历：队列先放根节点，每层先记录 queue.size()，循环 size 次取节点、记录值、加入左右孩子。
```

### 题解代码

```java
public List<List<Integer>> levelOrder(TreeNode root) {
    Queue<TreeNode> queue = new LinkedList<>();
    List<List<Integer>> result = new ArrayList<>();

    if (root != null) {
        queue.add(root);
    }

    while (!queue.isEmpty()) {
        int size = queue.size();
        List<Integer> level = new ArrayList<>();

        while (size-- > 0) {
            TreeNode node = queue.poll();
            level.add(node.val);

            if (node.left != null) {
                queue.add(node.left);
            }
            if (node.right != null) {
                queue.add(node.right);
            }
        }

        result.add(level);
    }

    return result;
}
```

### 快速理解

```text
        3
       / \
      9   20
         /  \
        15   7
```

队列变化：

| 当前层 | 层开始时队列 | `size` | 处理后加入的下一层节点 | 当前层结果 |
| --- | --- | --- | --- | --- |
| 第 1 层 | `[3]` | 1 | `[9, 20]` | `[3]` |
| 第 2 层 | `[9, 20]` | 2 | `[15, 7]` | `[9, 20]` |
| 第 3 层 | `[15, 7]` | 2 | `[]` | `[15, 7]` |

最终：

```text
[[3], [9, 20], [15, 7]]
```

### 外层和内层循环的职责

```java
while (!queue.isEmpty())
```

控制处理多少层：队列还有节点，就继续处理下一层。

```java
while (size-- > 0)
```

控制当前层处理多少节点：只处理本层开始时队列中的节点。

### 易错点

- `size` 必须在进入内层循环前保存；不能边遍历边使用不断变化的 `queue.size()`。
- 子节点加入队列后属于下一层，不能在当前层继续处理。
- 空树时队列为空，直接返回空列表。
- 队列取出的变量建议命名为 `node` 或 `current`，不要使用含义不清的名称。

### 复杂度

- 时间复杂度：`O(n)`，每个节点入队、出队各一次。
- 队列空间复杂度：`O(w)`，`w` 为树的最大宽度。
- 返回结果空间复杂度：`O(n)`。

## H_114 - 二叉树展开为链表

### 核心思路

题目要求将二叉树原地展开为一条只使用 `right` 指针的链表，节点顺序必须与前序遍历一致：

```text
根 -> 左子树 -> 右子树
```

使用反向前序遍历：

```text
右子树 -> 左子树 -> 根节点
```

成员变量 `prev` 保存当前节点在最终链表中的后继节点。处理当前节点时，直接将其接到 `prev` 前面：

```java
root.right = prev;
root.left = null;
prev = root;
```

### 面试速记

```text
反向前序遍历（右 -> 左 -> 根）：用 prev 记录已展开链表头，当前节点的 right 指向 prev，left 置空。
```

### 题解代码

```java
TreeNode prev = null;

public void flatten(TreeNode root) {
    prev = null;
    dfs(root);
}

public void dfs(TreeNode root) {
    if (root == null) {
        return;
    }

    dfs(root.right);
    dfs(root.left);

    root.right = prev;
    root.left = null;
    prev = root;
}
```

### 快速理解

原树：

```text
    1
   / \
  2   5
 / \   \
3   4   6
```

目标链表：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

递归实际处理顺序为：

```text
6 -> 5 -> 4 -> 3 -> 2 -> 1
```

拼接过程：

```text
处理 6：6.right = null
处理 5：5.right = 6
处理 4：4.right = 5
处理 3：3.right = 4
处理 2：2.right = 3
处理 1：1.right = 2
```

最终就是前序顺序：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

### 为什么先递归右子树

当前节点最终应连接到前序遍历中的下一个节点。先处理右子树、再处理左子树后，处理根节点时：

```text
prev = 左子树展开后的头节点
```

将：

```java
root.right = prev;
```

就能得到正确的前序连接顺序。

### 易错点

- 递归顺序必须是 `右 -> 左 -> 根`，不能直接按普通前序的 `根 -> 左 -> 右` 修改指针。
- 每个节点都要执行 `root.left = null`，题目要求展开后没有左指针。
- `prev` 是成员变量；若同一个 `Solution` 对象重复调用 `flatten`，入口处必须重置 `prev = null`。

### 复杂度

- 时间复杂度：`O(n)`，每个节点只访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。

## H_101 - 对称二叉树

### 核心思路

判断对称不是比较左右子树是否相同，而是判断它们是否互为镜像。

对于两个镜像位置的节点 `left`、`right`：

```text
left.left  <-> right.right
left.right <-> right.left
```

递归函数 `dfs(left, right)` 的含义是：判断这两个节点为根的子树是否镜像对称。

### 面试速记

```text
镜像 DFS：左的左对右的右、左的右对右的左；两个节点同时为空为 true，一个为空或值不同为 false。
```

这不是单节点的前中后序遍历，而是双节点成对递归。

### 题解代码

```java
public boolean isSymmetric(TreeNode root) {
    if (root == null) {
        return true;
    }
    return dfs(root.left, root.right);
}

public boolean dfs(TreeNode left, TreeNode right) {
    if (left == null || right == null) {
        return left == right;
    }

    if (left.val != right.val) {
        return false;
    }

    return dfs(left.left, right.right)
        && dfs(left.right, right.left);
}
```

### 快速理解

对称树：

```text
        1
       / \
      2   2
     / \ / \
    3  4 4  3
```

递归比较：

```text
dfs(2左, 2右)
dfs(3, 3)  // 左子树外侧 vs 右子树外侧
dfs(4, 4)  // 左子树内侧 vs 右子树内侧
```

每一对节点的结构和值都对应，因此返回 `true`。

反例：

```text
        1
       / \
      2   2
       \   \
        3   3
```

虽然两个 `3` 的值相同，但它们都在右侧，不是镜像位置。递归会比较：

```text
left.left = null
right.right = 3
```

一个为空、一个不为空，返回 `false`。

### 空节点判断

```java
if (left == null || right == null) {
    return left == right;
}
```

含义：

```text
两个都是 null：true
只有一个是 null：false
```

这等价于分别判断“两者都为空”和“只有一个为空”，但写法更简洁。

### 易错点

- 必须交叉递归：`left.left` 对 `right.right`，`left.right` 对 `right.left`。
- `dfs(left.left, right.left)` 这种写法判断的是两棵树相同，不是镜像。
- 入口处处理 `root == null`，避免访问 `root.left` 时空指针。
- 节点值相同还不够，节点位置和子树结构也必须镜像。

### 复杂度

- 时间复杂度：`O(n)`，每个节点最多访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。

## H_104 - 二叉树的最大深度

### 核心思路

使用后序递归。`dfs(root)` 的返回值定义为：以 `root` 为根的子树最大深度。

```text
当前树最大深度 = max(左子树最大深度, 右子树最大深度) + 1
```

其中 `+1` 表示当前节点本身。

### 面试速记

```text
后序遍历（左 -> 右 -> 根）：dfs 返回子树高度，当前高度等于 max(左高, 右高) + 1。
```

### 题解代码

```java
public int maxDepth(TreeNode root) {
    return dfs(root);
}

public int dfs(TreeNode root) {
    if (root == null) {
        return 0;
    }

    int leftHeight = dfs(root.left);
    int rightHeight = dfs(root.right);
    int maxHeight = Math.max(leftHeight, rightHeight) + 1;
    return maxHeight;
}
```

### 为什么空节点返回 `0`

空树不包含任何节点，所以深度为 `0`。

叶子节点左右孩子都为空：

```java
max(0, 0) + 1 = 1;
```

因此叶子节点深度为 `1`，符合从当前节点到自身经过一个节点的定义。

### 快速理解

示例：

```text
        3
       / \
      9   20
         /  \
        15   7
```

从叶子节点开始向上计算：

| 节点 | 左子树高度 | 右子树高度 | 当前子树高度 |
| --- | --- | --- | --- |
| `9` | 0 | 0 | 1 |
| `15` | 0 | 0 | 1 |
| `7` | 0 | 0 | 1 |
| `20` | 1 | 1 | 2 |
| `3` | 1 | 2 | 3 |

最终：

```text
maxDepth(3) = 3
```

最长路径可以是：

```text
3 -> 20 -> 15
```

### 和 H_543 的区别

两题的 `dfs` 都返回子树高度，但用途不同：

| 题目 | `dfs` 返回值 | 当前节点的计算 |
| --- | --- | --- |
| H_104 最大深度 | 当前子树最大高度 | `max(left, right) + 1` |
| H_543 二叉树直径 | 当前子树最大高度 | `left + right` 更新全局直径 |

### 易错点

- `dfs` 返回的是高度，不是从根节点累计的层数。
- 当前节点必须等左右子树结果都得到后再计算，因此是后序思路。
- 深度按节点数计算，叶子节点深度是 `1`，空节点深度是 `0`。

### 复杂度

- 时间复杂度：`O(n)`，每个节点只访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。

## H_226 - 翻转二叉树

### 核心思路

遍历每一个节点，并将当前节点的左右孩子交换。当前节点交换完成后，再递归翻转左右子树。

```text
当前节点 -> 交换左右孩子 -> 递归翻转左右子树
```

### 面试速记

```text
前序遍历：访问每个节点时立即交换 left 和 right，再递归翻转交换后的两棵子树。
```

当前代码使用前序遍历：

```text
根 -> 左 -> 右
```

前序不是唯一可行顺序，后序也可以；这版选择前序是因为“访问节点就交换孩子”的思路最直接。

### 题解代码

```java
public TreeNode invertTree(TreeNode root) {
    if (root == null) {
        return null;
    }

    swapChildren(root);
    invertTree(root.left);
    invertTree(root.right);
    return root;
}

private void swapChildren(TreeNode root) {
    TreeNode tmp = root.left;
    root.left = root.right;
    root.right = tmp;
}
```

### 快速理解

原树：

```text
        4
       / \
      2   7
     / \ / \
    1  3 6  9
```

先交换根节点：

```text
    4
   / \
  7   2
```

然后递归翻转节点 `7` 和节点 `2` 的子树，最终得到：

```text
        4
       / \
      7   2
     / \ / \
    9  6 3  1
```

### 为什么需要 `tmp`

交换左右孩子时必须先保存原来的左孩子：

```java
TreeNode tmp = root.left;
root.left = root.right;
root.right = tmp;
```

如果直接写：

```java
root.left = root.right;
root.right = root.left;
```

第二行读取到的 `root.left` 已经是原来的右孩子，原左子树会丢失。

### 易错点

- 空节点直接返回，避免访问 `root.left` 和 `root.right` 时空指针。
- 交换后递归处理 `root.left`、`root.right` 没有问题，它们只是交换后的两棵子树，都会被翻转一次。
- 当前写法会原地修改输入树，返回值仍然是原根节点 `root`。
- 先递归、后交换也可以；关键是每个节点的左右孩子都恰好交换一次。

### 复杂度

- 时间复杂度：`O(n)`，每个节点只访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。

## H_236 - 二叉树的最近公共祖先

### 核心思路

使用后序递归，分别在左右子树查找 `p` 和 `q`。如果左右递归结果都非空，说明 `p`、`q` 分别位于当前节点的两侧，当前节点就是最近公共祖先。

递归函数的返回含义：

```text
在以 root 为根的子树中：
- 找到 p 或 q：返回该节点。
- 同时找到 p 和 q：返回它们的最近公共祖先。
- 两者都找不到：返回 null。
```

### 面试速记

```text
后序遍历（左 -> 右 -> 根）：左右各找 p/q，左右都非空则当前节点是 LCA，否则返回非空的一边。
```

### 题解代码

```java
public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) {
        return root;
    }

    TreeNode left = lowestCommonAncestor(root.left, p, q);
    TreeNode right = lowestCommonAncestor(root.right, p, q);

    if (left != null && right != null) {
        return root;
    }

    return left != null ? left : right;
}
```

### 快速理解

示例：

```text
        3
       / \
      5   1
     / \ / \
    6  2 0  8
      / \
     7   4
```

若：

```text
p = 5, q = 1
```

在节点 `3`：

```text
left = 5
right = 1
```

左右都非空，所以：

```java
return root; // 3
```

若：

```text
p = 5, q = 4
```

递归到节点 `5` 时，`root == p`，直接返回 `5`。由于 `5` 也是 `4` 的祖先，它就是最近公共祖先。

### 四种返回情况

| `left` | `right` | 含义 | 返回值 |
| --- | --- | --- | --- |
| 非空 | 非空 | `p`、`q` 分别在左右子树 | `root` |
| 非空 | 空 | 两个目标都在左侧，或当前只在左侧找到目标 | `left` |
| 空 | 非空 | 两个目标都在右侧，或当前只在右侧找到目标 | `right` |
| 空 | 空 | 当前子树没有 `p`、`q` | `null` |

### 易错点

- 判断目标节点要比较引用：`root == p || root == q`，不能比较节点值；节点值可能重复。
- 当前节点本身是 `p` 或 `q` 时直接返回，因为它可能是另一个节点的祖先。
- 题目保证 `p`、`q` 都在树中；若不保证，需要额外记录是否两个节点都找到。
- 不能只判断根节点，最近公共祖先可能在任意子树内部。

### 复杂度

- 时间复杂度：`O(n)`，每个节点最多访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。

## H_617 - 合并二叉树

### 核心思路

递归合并两棵树的同位置节点：

```text
root1.left  <-> root2.left
root1.right <-> root2.right
```

规则：

- 两个节点都存在：创建新节点，节点值为两者之和，再递归合并左右孩子。
- 其中一个节点为空：直接返回另一个节点，该位置无需相加。

### 面试速记

```text
前序递归（根 -> 左 -> 右）：同位置节点相加后递归合并左右孩子；一方为空直接返回另一方。
```

### 题解代码

```java
public TreeNode mergeTrees(TreeNode root1, TreeNode root2) {
    if (root1 == null) {
        return root2;
    }
    if (root2 == null) {
        return root1;
    }

    TreeNode root = new TreeNode();
    root.val = root1.val + root2.val;
    root.left = mergeTrees(root1.left, root2.left);
    root.right = mergeTrees(root1.right, root2.right);
    return root;
}
```

### 快速理解

```text
Tree 1:        1          Tree 2:        2
              / \\                       / \
             3   2                     1   3
            /                           \\   \
           5                             4   7
```

从根节点开始：

```text
1 + 2 = 3
3 + 1 = 4
2 + 3 = 5
```

合并结果：

```text
        3
       / \
      4   5
     / \\   \
    5   4   7
```

左子树中的 `5` 只存在于 Tree 1，因此直接返回该节点；右子树中的 `4`、`7` 只存在于 Tree 2，也直接返回对应节点。

### 递归函数含义

```text
mergeTrees(root1, root2)
返回以 root1 和 root2 为根的两棵子树合并后的根节点。
```

两个节点都存在时，先合并当前值：

```java
root.val = root1.val + root2.val;
```

再递归合并左右同位置的孩子：

```java
root.left = mergeTrees(root1.left, root2.left);
root.right = mergeTrees(root1.right, root2.right);
```

### 易错点

- 合并的是同一位置的节点，不能交叉递归左右子树。
- 当前写法仅在两个节点都存在时新建节点；某一侧为空时直接返回另一棵树的节点。
- 因此结果树会复用输入树中未重叠部分的节点引用。LeetCode 可以接受；若要求结果树完全独立，需要深拷贝该子树。
- 两个节点都为 `null` 时，第一个 `if` 会返回 `null`，无需额外处理。

### 复杂度

- 时间复杂度：`O(m + n)`，每个存在的节点最多处理一次。
- 空间复杂度：`O(h)`，递归栈深度为两棵树的最大高度；最坏为 `O(m + n)`。

## H_437 - 路径总和 III

### 核心思路

使用前缀和 + DFS 回溯，统计所有“向下且路径和为 `targetSum`”的路径。路径不要求从根开始，也不要求到叶子结束。

定义：

```text
curSum = 从根节点到当前节点的前缀和
```

如果某段以当前节点为终点的路径和为 `targetSum`，则需要某个历史前缀和：

```text
curSum - 历史前缀和 = targetSum
历史前缀和 = curSum - targetSum
```

`prefixSumCount` 记录当前根到当前节点路径上，每种前缀和出现的次数。

### 面试速记

```text
前序 DFS + 回溯：加当前值，查 curSum - target，加入当前前缀和，递归左右，离开节点时撤销当前前缀和。
```

### 题解代码

```java
public int pathSum(TreeNode root, int targetSum) {
    Map<Long, Integer> prefixSumCount = new HashMap<>();
    prefixSumCount.put(0L, 1);
    return dfs(root, 0L, targetSum, prefixSumCount);
}

public int dfs(TreeNode node, long curSum, int targetSum,
               Map<Long, Integer> prefixSumCount) {
    if (node == null) {
        return 0;
    }

    curSum += node.val;

    long neededPrefixSum = curSum - targetSum;
    int pathCount = prefixSumCount.getOrDefault(neededPrefixSum, 0);

    prefixSumCount.put(curSum, prefixSumCount.getOrDefault(curSum, 0) + 1);

    pathCount += dfs(node.left, curSum, targetSum, prefixSumCount);
    pathCount += dfs(node.right, curSum, targetSum, prefixSumCount);

    prefixSumCount.put(curSum, prefixSumCount.get(curSum) - 1);

    return pathCount;
}
```

### 为什么初始化 `0L -> 1`

```java
prefixSumCount.put(0L, 1);
```

表示根节点之前存在一个前缀和为 `0` 的虚拟位置。

当从根到当前节点的路径和刚好等于目标值时：

```text
curSum - targetSum = 0
```

Map 中的 `0 -> 1` 就能统计从根节点开始的这条路径。

### 图解：路径 `10 -> 5 -> 3`

目标值：

```text
targetSum = 8
```

树的一部分：

```text
    10
   /
  5
 /
3
```

前缀和变化：

| 当前节点 | `curSum` | 需要查找 `curSum - 8` | 查询前 Map | 找到的路径 |
| --- | --- | --- | --- | --- |
| 虚拟起点 | 0 | - | `{0: 1}` | - |
| `10` | 10 | 2 | `{0: 1}` | 无 |
| `5` | 15 | 7 | `{0: 1, 10: 1}` | 无 |
| `3` | 18 | 10 | `{0: 1, 10: 1, 15: 1}` | `5 -> 3` |

走到节点 `3` 时：

```text
18 - 10 = 8
```

历史前缀和 `10` 对应根节点 `10`，因此从它的下一个节点到当前节点：

```text
5 -> 3
```

路径和正好为 `8`。

### 为什么必须回溯

进入节点时：

```java
prefixSumCount.put(curSum, prefixSumCount.getOrDefault(curSum, 0) + 1);
```

离开节点时：

```java
prefixSumCount.put(curSum, prefixSumCount.get(curSum) - 1);
```

Map 只能保存当前递归路径的前缀和，不能让左子树的状态影响右子树。

```text
      10
     /  \
    5   -3
```

访问右子树 `-3` 时，节点 `5` 不在路径 `10 -> -3` 上；不回溯会把跨左右子树的错误路径算进去。

### 易错点

- 先查询 `curSum - targetSum`，再将当前 `curSum` 放入 Map；否则 `targetSum = 0` 时可能把空路径误计入。
- Map 的 value 是出现次数，不能只存是否存在；同一个前缀和可能在一条路径中出现多次。
- 使用 `long curSum` 和 `Map<Long, Integer>`，避免路径和累加导致 `int` 溢出。
- 回溯时只减 1 即可，不一定要从 Map 删除 value 为 `0` 的 key。

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次。
- 空间复杂度：`O(h)`，Map 和递归栈只保存当前路径状态；最坏为 `O(n)`。

## H_538 - 把二叉搜索树转换为累加树

### 核心思路

BST 的中序遍历是从小到大：

```text
左 -> 根 -> 右
```

题目需要每个节点累加所有更大节点的值，因此使用反向中序遍历：

```text
右 -> 根 -> 左
```

这样节点按从大到小的顺序处理。成员变量 `prev` 保存已经处理过的所有更大节点的累计和。

### 面试速记

```text
BST 反向中序（右 -> 根 -> 左）：prev 保存已访问更大节点的累计和，当前节点加 prev 后更新 prev。
```

### 题解代码

```java
int prev = 0;

public TreeNode convertBST(TreeNode root) {
    prev = 0;
    dfs(root);
    return root;
}

public void dfs(TreeNode root) {
    if (root == null) {
        return;
    }

    dfs(root.right);
    root.val += prev;
    prev = root.val;
    dfs(root.left);
}
```

### `prev` 的含义

`prev` 不是前一个节点的原始值，而是：

```text
当前节点之前已经访问过的所有更大节点的累计和。
```

处理当前节点时：

```java
root.val += prev;
```

得到“当前节点值 + 所有更大节点值之和”。随后：

```java
prev = root.val;
```

当前节点更新后的累计结果会提供给更小的节点使用。

### 快速理解

原 BST：

```text
      4
     / \
    1   6
       / \
      5   7
```

反向中序访问顺序：

```text
7 -> 6 -> 5 -> 4 -> 1
```

| 当前节点 | 处理前 `prev` | 更新后的节点值 | 处理后 `prev` |
| --- | --- | --- | --- |
| `7` | 0 | `7 + 0 = 7` | 7 |
| `6` | 7 | `6 + 7 = 13` | 13 |
| `5` | 13 | `5 + 13 = 18` | 18 |
| `4` | 18 | `4 + 18 = 22` | 22 |
| `1` | 22 | `1 + 22 = 23` | 23 |

转换后：

```text
      22
     /  \
    23   13
        /  \
       18   7
```

### 为什么先访问右子树

当前节点需要知道所有更大节点的和。BST 的右子树全部大于当前节点，所以先处理右子树后，`prev` 已经包含所有需要累加的更大值。

普通中序是从小到大，处理当前节点时更大的节点还未访问，无法直接完成累加。

### 易错点

- 必须是 `右 -> 根 -> 左`，不能使用普通中序 `左 -> 根 -> 右`。
- `root.val` 更新后已经是累计和，因此要写 `prev = root.val`，不能再写 `prev += root.val`。
- `prev` 是成员变量；入口处重置 `prev = 0`，避免同一个 `Solution` 对象多次调用时串值。
- 该写法原地修改节点值，不会创建新树或改变树结构。

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。

## H_543 - 二叉树的直径

### 核心思路

二叉树的直径是任意两个节点之间最长路径的边数，路径不一定经过根节点。

这题使用后序遍历：先计算左右子树高度，再计算经过当前节点的最长路径。

```text
dfs 的返回值：当前节点向下的最大高度
maxDiameter：遍历过程中记录的最大直径
```

### 面试速记

```text
后序遍历（左 -> 右 -> 根）：dfs 返回子树高度，每个节点用 left + right 更新经过该节点的最大直径。
```

### 题解代码

```java
int maxDiameter = 0;

public int diameterOfBinaryTree(TreeNode root) {
    maxDiameter = 0;
    dfs(root);
    return maxDiameter;
}

public int dfs(TreeNode root) {
    if (root == null) {
        return 0;
    }

    int left = dfs(root.left);
    int right = dfs(root.right);

    maxDiameter = Math.max(maxDiameter, left + right);
    return Math.max(left, right) + 1;
}
```

### `dfs` 返回什么

`dfs(root)` 返回的不是直径，而是从当前节点向下走到叶子节点的最长路径包含的节点数。

例如：

```text
  2
 / \
4   5
```

```text
dfs(4) = 1
dfs(5) = 1
dfs(2) = max(1, 1) + 1 = 2
```

`dfs(2) = 2` 表示从节点 `2` 向下最长可经过两个节点，例如 `2 -> 4`。

### 快速理解

示例：

```text
        1
       / \
      2   3
     / \
    4   5
```

最长路径：

```text
4 -> 2 -> 1 -> 3
```

包含 4 个节点、3 条边，因此答案是 `3`。

后序计算过程：

| 当前节点 | `left` | `right` | 经过当前节点的路径边数 | `dfs` 返回高度 |
| --- | --- | --- | --- | --- |
| `4` | 0 | 0 | 0 | 1 |
| `5` | 0 | 0 | 0 | 1 |
| `2` | 1 | 1 | 2 | 2 |
| `3` | 0 | 0 | 0 | 1 |
| `1` | 2 | 1 | 3 | 3 |

处理节点 `1` 时：

```java
left + right = 2 + 1 = 3;
```

这正是路径 `4 -> 2 -> 1 -> 3` 的边数，因此更新最终答案。

### 为什么是 `left + right`

`left` 和 `right` 分别表示当前节点向左、向右最远能到的高度。

经过当前节点的最长路径为：

```text
左侧最深叶子 -> 当前节点 -> 右侧最深叶子
```

路径共有 `left + right + 1` 个节点，换算成边数要减 1：

```text
left + right + 1 - 1 = left + right
```

### 易错点

- `dfs` 返回高度，不能返回直径；父节点需要高度继续计算。
- 每个节点都可能是最长路径的中转点，因此每个节点都要用 `left + right` 更新 `maxDiameter`。
- 题目统计边数，不是节点数，所以更新答案时不要写 `left + right + 1`。
- `maxDiameter` 是成员变量；同一个对象重复调用时，应在入口重置为 `0`。

### 复杂度

- 时间复杂度：`O(n)`，每个节点只访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。
