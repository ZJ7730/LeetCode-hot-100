## H_114 - 二叉树展开为链表

### 核心思路

题目要求将二叉树原地展开为一条只使用 `right` 指针的链表，节点顺序必须与前序遍历一致：

```text
根 -> 左子树 -> 右子树
```

使用反向前序遍历：

```text
右子树 -> 左子树 -> 根节点
```

成员变量 `prev` 保存当前节点在最终链表中的后继节点。处理当前节点时，直接将其接到 `prev` 前面：

```java
root.right = prev;
root.left = null;
prev = root;
```

### 题解代码

```java
TreeNode prev = null;

public void flatten(TreeNode root) {
    prev = null;
    dfs(root);
}

public void dfs(TreeNode root) {
    if (root == null) {
        return;
    }

    dfs(root.right);
    dfs(root.left);

    root.right = prev;
    root.left = null;
    prev = root;
}
```

### 快速理解

原树：

```text
    1
   / \
  2   5
 / \   \
3   4   6
```

目标链表：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

递归实际处理顺序为：

```text
6 -> 5 -> 4 -> 3 -> 2 -> 1
```

拼接过程：

```text
处理 6：6.right = null
处理 5：5.right = 6
处理 4：4.right = 5
处理 3：3.right = 4
处理 2：2.right = 3
处理 1：1.right = 2
```

最终就是前序顺序：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

### 为什么先递归右子树

当前节点最终应连接到前序遍历中的下一个节点。先处理右子树、再处理左子树后，处理根节点时：

```text
prev = 左子树展开后的头节点
```

将：

```java
root.right = prev;
```

就能得到正确的前序连接顺序。

### 易错点

- 递归顺序必须是 `右 -> 左 -> 根`，不能直接按普通前序的 `根 -> 左 -> 右` 修改指针。
- 每个节点都要执行 `root.left = null`，题目要求展开后没有左指针。
- `prev` 是成员变量；若同一个 `Solution` 对象重复调用 `flatten`，入口处必须重置 `prev = null`。

### 复杂度

- 时间复杂度：`O(n)`，每个节点只访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。

## H_101 - 对称二叉树

### 核心思路

判断对称不是比较左右子树是否相同，而是判断它们是否互为镜像。

对于两个镜像位置的节点 `left`、`right`：

```text
left.left  <-> right.right
left.right <-> right.left
```

递归函数 `dfs(left, right)` 的含义是：判断这两个节点为根的子树是否镜像对称。

### 题解代码

```java
public boolean isSymmetric(TreeNode root) {
    if (root == null) {
        return true;
    }
    return dfs(root.left, root.right);
}

public boolean dfs(TreeNode left, TreeNode right) {
    if (left == null || right == null) {
        return left == right;
    }

    if (left.val != right.val) {
        return false;
    }

    return dfs(left.left, right.right)
        && dfs(left.right, right.left);
}
```

### 快速理解

对称树：

```text
        1
       / \
      2   2
     / \ / \
    3  4 4  3
```

递归比较：

```text
dfs(2左, 2右)
dfs(3, 3)  // 左子树外侧 vs 右子树外侧
dfs(4, 4)  // 左子树内侧 vs 右子树内侧
```

每一对节点的结构和值都对应，因此返回 `true`。

反例：

```text
        1
       / \
      2   2
       \   \
        3   3
```

虽然两个 `3` 的值相同，但它们都在右侧，不是镜像位置。递归会比较：

```text
left.left = null
right.right = 3
```

一个为空、一个不为空，返回 `false`。

### 空节点判断

```java
if (left == null || right == null) {
    return left == right;
}
```

含义：

```text
两个都是 null：true
只有一个是 null：false
```

这等价于分别判断“两者都为空”和“只有一个为空”，但写法更简洁。

### 易错点

- 必须交叉递归：`left.left` 对 `right.right`，`left.right` 对 `right.left`。
- `dfs(left.left, right.left)` 这种写法判断的是两棵树相同，不是镜像。
- 入口处处理 `root == null`，避免访问 `root.left` 时空指针。
- 节点值相同还不够，节点位置和子树结构也必须镜像。

### 复杂度

- 时间复杂度：`O(n)`，每个节点最多访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。

## H_104 - 二叉树的最大深度

### 核心思路

使用后序递归。`dfs(root)` 的返回值定义为：以 `root` 为根的子树最大深度。

```text
当前树最大深度 = max(左子树最大深度, 右子树最大深度) + 1
```

其中 `+1` 表示当前节点本身。

### 题解代码

```java
public int maxDepth(TreeNode root) {
    return dfs(root);
}

public int dfs(TreeNode root) {
    if (root == null) {
        return 0;
    }

    int leftHeight = dfs(root.left);
    int rightHeight = dfs(root.right);
    int maxHeight = Math.max(leftHeight, rightHeight) + 1;
    return maxHeight;
}
```

### 为什么空节点返回 `0`

空树不包含任何节点，所以深度为 `0`。

叶子节点左右孩子都为空：

```java
max(0, 0) + 1 = 1;
```

因此叶子节点深度为 `1`，符合从当前节点到自身经过一个节点的定义。

### 快速理解

示例：

```text
        3
       / \
      9   20
         /  \
        15   7
```

从叶子节点开始向上计算：

| 节点 | 左子树高度 | 右子树高度 | 当前子树高度 |
| --- | --- | --- | --- |
| `9` | 0 | 0 | 1 |
| `15` | 0 | 0 | 1 |
| `7` | 0 | 0 | 1 |
| `20` | 1 | 1 | 2 |
| `3` | 1 | 2 | 3 |

最终：

```text
maxDepth(3) = 3
```

最长路径可以是：

```text
3 -> 20 -> 15
```

### 和 H_543 的区别

两题的 `dfs` 都返回子树高度，但用途不同：

| 题目 | `dfs` 返回值 | 当前节点的计算 |
| --- | --- | --- |
| H_104 最大深度 | 当前子树最大高度 | `max(left, right) + 1` |
| H_543 二叉树直径 | 当前子树最大高度 | `left + right` 更新全局直径 |

### 易错点

- `dfs` 返回的是高度，不是从根节点累计的层数。
- 当前节点必须等左右子树结果都得到后再计算，因此是后序思路。
- 深度按节点数计算，叶子节点深度是 `1`，空节点深度是 `0`。

### 复杂度

- 时间复杂度：`O(n)`，每个节点只访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。

## H_617 - 合并二叉树

### 核心思路

递归合并两棵树的同位置节点：

```text
root1.left  <-> root2.left
root1.right <-> root2.right
```

规则：

- 两个节点都存在：创建新节点，节点值为两者之和，再递归合并左右孩子。
- 其中一个节点为空：直接返回另一个节点，该位置无需相加。

### 题解代码

```java
public TreeNode mergeTrees(TreeNode root1, TreeNode root2) {
    if (root1 == null) {
        return root2;
    }
    if (root2 == null) {
        return root1;
    }

    TreeNode root = new TreeNode();
    root.val = root1.val + root2.val;
    root.left = mergeTrees(root1.left, root2.left);
    root.right = mergeTrees(root1.right, root2.right);
    return root;
}
```

### 快速理解

```text
Tree 1:        1          Tree 2:        2
              / \\                       / \
             3   2                     1   3
            /                           \\   \
           5                             4   7
```

从根节点开始：

```text
1 + 2 = 3
3 + 1 = 4
2 + 3 = 5
```

合并结果：

```text
        3
       / \
      4   5
     / \\   \
    5   4   7
```

左子树中的 `5` 只存在于 Tree 1，因此直接返回该节点；右子树中的 `4`、`7` 只存在于 Tree 2，也直接返回对应节点。

### 递归函数含义

```text
mergeTrees(root1, root2)
返回以 root1 和 root2 为根的两棵子树合并后的根节点。
```

两个节点都存在时，先合并当前值：

```java
root.val = root1.val + root2.val;
```

再递归合并左右同位置的孩子：

```java
root.left = mergeTrees(root1.left, root2.left);
root.right = mergeTrees(root1.right, root2.right);
```

### 易错点

- 合并的是同一位置的节点，不能交叉递归左右子树。
- 当前写法仅在两个节点都存在时新建节点；某一侧为空时直接返回另一棵树的节点。
- 因此结果树会复用输入树中未重叠部分的节点引用。LeetCode 可以接受；若要求结果树完全独立，需要深拷贝该子树。
- 两个节点都为 `null` 时，第一个 `if` 会返回 `null`，无需额外处理。

### 复杂度

- 时间复杂度：`O(m + n)`，每个存在的节点最多处理一次。
- 空间复杂度：`O(h)`，递归栈深度为两棵树的最大高度；最坏为 `O(m + n)`。

## H_543 - 二叉树的直径

### 核心思路

二叉树的直径是任意两个节点之间最长路径的边数，路径不一定经过根节点。

这题使用后序遍历：先计算左右子树高度，再计算经过当前节点的最长路径。

```text
dfs 的返回值：当前节点向下的最大高度
maxDiameter：遍历过程中记录的最大直径
```

### 题解代码

```java
int maxDiameter = 0;

public int diameterOfBinaryTree(TreeNode root) {
    maxDiameter = 0;
    dfs(root);
    return maxDiameter;
}

public int dfs(TreeNode root) {
    if (root == null) {
        return 0;
    }

    int left = dfs(root.left);
    int right = dfs(root.right);

    maxDiameter = Math.max(maxDiameter, left + right);
    return Math.max(left, right) + 1;
}
```

### `dfs` 返回什么

`dfs(root)` 返回的不是直径，而是从当前节点向下走到叶子节点的最长路径包含的节点数。

例如：

```text
  2
 / \
4   5
```

```text
dfs(4) = 1
dfs(5) = 1
dfs(2) = max(1, 1) + 1 = 2
```

`dfs(2) = 2` 表示从节点 `2` 向下最长可经过两个节点，例如 `2 -> 4`。

### 快速理解

示例：

```text
        1
       / \
      2   3
     / \
    4   5
```

最长路径：

```text
4 -> 2 -> 1 -> 3
```

包含 4 个节点、3 条边，因此答案是 `3`。

后序计算过程：

| 当前节点 | `left` | `right` | 经过当前节点的路径边数 | `dfs` 返回高度 |
| --- | --- | --- | --- | --- |
| `4` | 0 | 0 | 0 | 1 |
| `5` | 0 | 0 | 0 | 1 |
| `2` | 1 | 1 | 2 | 2 |
| `3` | 0 | 0 | 0 | 1 |
| `1` | 2 | 1 | 3 | 3 |

处理节点 `1` 时：

```java
left + right = 2 + 1 = 3;
```

这正是路径 `4 -> 2 -> 1 -> 3` 的边数，因此更新最终答案。

### 为什么是 `left + right`

`left` 和 `right` 分别表示当前节点向左、向右最远能到的高度。

经过当前节点的最长路径为：

```text
左侧最深叶子 -> 当前节点 -> 右侧最深叶子
```

路径共有 `left + right + 1` 个节点，换算成边数要减 1：

```text
left + right + 1 - 1 = left + right
```

### 易错点

- `dfs` 返回高度，不能返回直径；父节点需要高度继续计算。
- 每个节点都可能是最长路径的中转点，因此每个节点都要用 `left + right` 更新 `maxDiameter`。
- 题目统计边数，不是节点数，所以更新答案时不要写 `left + right + 1`。
- `maxDiameter` 是成员变量；同一个对象重复调用时，应在入口重置为 `0`。

### 复杂度

- 时间复杂度：`O(n)`，每个节点只访问一次。
- 空间复杂度：`O(h)`，递归栈深度为树高 `h`；最坏为 `O(n)`。
