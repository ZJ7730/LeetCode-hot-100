## 4. 易错点

- 网格 BFS/DFS 要先检查边界，再访问数组。
- 遍历过的位置要及时标记，避免重复入队或递归死循环。
- 多源 BFS 要先把所有起点入队，再开始按层扩散。
- 拓扑排序最后要判断是否处理了全部节点。

## 5. 每题笔记

### H_200 - 岛屿数量

- 题目定位：在 `m * n` 网格中统计岛屿数量，`'1'` 表示陆地，`'0'` 表示水域。
- 当前代码思路：外层双重循环扫描所有格子，遇到未访问过的陆地就把答案加一，再用 DFS 把这一整座岛屿全部标记为已访问。
- 关键变量：`visited` 记录格子是否已经被搜索过；`res` 记录岛屿数量；`x` 和 `y` 表示 DFS 当前搜索位置。
- 解题步骤：扫描网格时，如果 `!visited[i][j] && grid[i][j] == '1'`，说明发现新岛屿；调用 `dfs` 后，这座岛屿上下左右连通的陆地都会被标记，后续不会重复计数。
- DFS 终止条件：坐标越界时直接返回；当前格子已经访问过或是水域时也直接返回。
- 注意事项：DFS 中一定要先判断边界，再访问数组；标记 `visited[x][y] = true` 要放在继续递归四个方向之前，避免重复递归。
- 复杂度：时间 `O(m * n)`，每个格子最多处理一次；空间 `O(m * n)`，来自 `visited` 和递归栈。
- 面试易错点：岛屿数量是在发现“未访问陆地”时加一，不是在 DFS 每访问一个陆地时加一。
- 可复述话术：我从网格中找每座岛屿的入口，找到一个未访问陆地就计数一次，然后用 DFS 把它连着的所有陆地淹掉，避免后续重复统计。

### H_207 - 课程表

- 题目定位：给定课程数量和先修关系，判断是否可以学完所有课程。课程依赖可以抽象成有向图，如果图里存在环，就会出现互相等待的课程，无法完成学习。
- 当前代码思路：使用 BFS 拓扑排序。先根据先修关系建立邻接表和入度表，再从所有入度为 `0` 的课程开始处理，模拟“先学没有前置依赖的课，再解锁后续课程”的过程。
- 为什么能判断环：没有环的有向图一定能不断找到入度为 `0` 的节点；如果存在环，环里的课程都至少依赖环中的另一门课，入度无法被减到 `0`，最后就不会被处理到。
- 建图方向：`prerequisites` 中 `[cur, pre]` 表示学习 `cur` 前必须先学 `pre`，所以有向边是 `pre -> cur`。代码里的 `graph.get(pre).add(cur)` 表示学完 `pre` 后，`cur` 的一个前置条件被满足。
- 关键变量：`graph[i]` 表示学完课程 `i` 后可以继续学习哪些课程；`indegree[i]` 表示课程 `i` 当前还剩多少前置课程；`queue` 保存当前所有入度为 `0`、可以直接学习的课程；`res` 记录已经成功学习的课程数。
- 解题步骤：初始化每门课的后续课程列表；遍历 `prerequisites` 建图并统计入度；把所有入度为 `0` 的课程加入队列；每次弹出一门课表示学完它，然后遍历它能解锁的后续课程，把这些课程的入度减一；如果某门后续课程入度减到 `0`，说明它的前置课程全部完成，可以继续入队。
- 判断方式：如果最终 `res == numCourses`，说明所有课程都能按拓扑顺序学完；如果 `res < numCourses`，说明剩余课程的入度一直无法归零，存在环形依赖。
- 注意事项：邻接表存的是“当前课程能解锁哪些课”，不是“当前课程依赖哪些课”；`[cur, pre]` 的边方向不能写反；最后要比较处理过的课程数是否等于总课程数，不能只看队列是否为空。
- 复杂度：时间 `O(numCourses + prerequisites.length)`，每门课程和每条依赖最多处理一次；空间 `O(numCourses + prerequisites.length)`，主要是邻接表、入度数组和队列。
- 面试易错点：`res++` 代表成功学习了一门课，不是依赖数量；入度减一代表完成了一个前置条件，不代表这门课一定马上可学，只有入度变成 `0` 才能入队。
- 可复述话术：我把课程看成有向图，先学入度为零的课，每学完一门课就减少后续课程的入度；如果最后所有课都被处理，说明没有环，可以学完。

### H_994 - 腐烂的橘子

- 题目定位：网格中腐烂橘子会每分钟向四周扩散，求所有新鲜橘子都腐烂需要的最少分钟数。
- 当前代码思路：把所有初始腐烂橘子同时入队，进行多源 BFS，按层向外扩散。
- 关键变量：`fresh` 记录剩余新鲜橘子数；队列中保存当前层的腐烂橘子坐标；`minutes` 记录经过的分钟数。
- 解题步骤：先统计新鲜橘子并把初始腐烂橘子入队；每轮处理一层队列表示过了一分钟；如果相邻新鲜橘子被感染，就标记为腐烂并入队。
- 复杂度：时间 `O(rows * cols)`，空间 `O(rows * cols)`。
- 面试易错点：多源 BFS 必须先把所有初始腐烂橘子一起入队，不能一个一个单独扩散。
- 可复述话术：我把所有初始腐烂橘子当成同一层起点，层序扩散代表时间流逝，最后看是否还有新鲜橘子没被感染。
