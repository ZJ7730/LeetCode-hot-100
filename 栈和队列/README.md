## 4. 易错点

- 栈题要先确定栈中保存字符、下标、状态还是中间结果。

- 双堆要保持大小差不超过 1，并保证左半部分都小于右半部分。

## 5. 每题笔记

### H_20 - 有效的括号

- 题目定位：判断括号字符串是否合法。
- 当前代码思路：用栈保存当前期待出现的右括号，遇到左括号就把对应右括号压入栈。
- 关键变量：栈顶是当前最应该匹配的字符。
- 解题步骤：遍历字符串；左括号入栈对应右括号；右括号则检查是否与栈顶匹配；最后看栈是否为空。
- 复杂度：时间 `O(n)`，空间 `O(n)`。
- 面试易错点：不要把“括号本身”压栈和“期待的右括号”混淆，当前写法是直接压入期待字符。
- 可复述话术：我让栈顶始终保存下一个应该出现的右括号，这样遇到右括号时只要看栈顶是否一致即可。

### H_155 - 最小栈

- 题目定位：设计一个栈，支持 `push/pop/top/getMin` 都在常数时间内完成。
- 当前代码思路：维护两个栈，一个存全部元素，一个同步记录当前最小值。
- 关键变量：`stack` 存数据；`minStack` 存当前最小值序列。
- 解题步骤：push 时如果新值不大于最小栈栈顶，就同步压入最小栈；pop 时如果弹出值等于最小栈栈顶，也同步弹出。
- 复杂度：所有操作都是 O(1)。
- 面试易错点：最小栈不能只记录一个最小值，因为弹出后最小值可能回退到之前的次小值。
- 可复述话术：主栈负责正常出入栈，辅助栈负责同步保存每个阶段的最小值。

### H_239 - 滑动窗口最大值

- 题目定位：求每个长度为 `k` 的窗口中的最大值。
- 当前代码思路：用单调队列维护窗口内从大到小的候选值，队首始终是当前最大值。
- 关键变量：`queue` 保存候选最大值；`getMax()` 返回队首。
- 解题步骤：初始化前 `k` 个数；每次滑动时先把滑出窗口的值弹出，再把新值压入；读取队首作为当前答案。
- 复杂度：时间 `O(n)`，空间 `O(k)`。
- 面试易错点：单调队列里保存的是值而不是下标时，出队只能通过比较窗口滑出的值和队首判断。
- 可复述话术：我用一个从大到小排列的队列维护窗口中的候选最大值，队首永远是当前窗口的最大元素。

### H_295 - 数据流的中位数

- 题目定位：支持动态插入数字并实时返回中位数。
- 当前代码思路：用两个堆维护左右两半数据，左边是大顶堆，右边是小顶堆。
- 关键变量：`small` 存较小的一半；`large` 存较大的一半。
- 解题步骤：插入时先放入合适的堆；再平衡两堆大小，使两边数量差不超过 1；取中位数时根据堆大小关系决定返回一个数还是平均值。
- 复杂度：插入 O(log n)，查询中位数 O(1)。
- 面试易错点：两个堆不仅要平衡数量，还要保持左半部分都小于右半部分。
- 可复述话术：我把数据流拆成左右两半，左边最大堆保存较小部分，右边最小堆保存较大部分，中位数就能随时读出来。
