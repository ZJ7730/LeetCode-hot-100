## 2. 本专题核心思想

- 回溯就是在递归树上做选择、进入下一层、撤销选择。
- `path` 表示当前路径，`result` 表示所有合法答案。
- 组合问题靠 `startIndex` 防止重复选择之前的元素。
- 排列问题没有 `startIndex`，靠 `used[]` 判断当前路径是否已经使用过某个元素。
- 去重先想清楚是树层去重还是树枝去重：同一层不要选重复值，路径内部可以根据题意决定是否允许重复。
- 剪枝的本质是提前停止不可能产生答案的分支。

## 5. 每题笔记

### H_77 - 组合

- 题目定位：从 `1..n` 中选择 `k` 个数。
- 当前代码思路：用 `startIndex` 控制每层递归的起点，避免重复组合。
- 关键变量：`path` 保存当前组合；`path.size() == k` 时收集结果。
- 解题步骤：枚举当前层可选数字；加入 `path`；递归下一层；撤销选择。
- 复杂度：时间 `O(k * C(n,k))`，空间 `O(k)`。
- 面试易错点：每层递归必须从当前数字后面继续选，不能再回到前面。
- 可复述话术：组合不关心顺序，所以我用 `startIndex` 保证每次只向后选。

### H_77_剪枝 - 组合，剪枝版

- 题目定位：`H_77` 的优化版本。
- 当前代码思路：根据剩余需要选择的数量，收缩当前层循环右边界。
- 关键变量：`n - (k - path.size()) + 1` 是当前层最大可选值。
- 解题步骤：先判断是否已选够；再按剪枝边界枚举；递归和回溯。
- 复杂度：结果规模不变，但减少无效分支。
- 面试易错点：右边界是闭区间，因此要加 `1`。
- 可复述话术：如果后面剩余的位置不够填满答案，就没有必要继续枚举。

### H_216 - 组合总和 III

- 题目定位：从 `1..9` 中选 `k` 个数，使和为 `n`。
- 当前代码思路：在组合模板上加 `sum` 和固定范围约束。
- 关键变量：`sum` 记录当前路径总和；`path.size()` 记录已选数量。
- 解题步骤：枚举 `1..9`；加入数字并累加；递归 `i + 1`；回溯恢复。
- 复杂度：和有效组合数量相关，空间 `O(k)`。
- 面试易错点：必须同时满足数量和总和两个条件。
- 可复述话术：这是固定候选范围的组合题，数量和总和一起剪枝。

### H_39 - 组合总和

- 题目定位：候选数字可以重复使用，找出所有和为 `target` 的组合。
- 当前代码思路：组合回溯中，递归时继续传当前下标，让当前数字可以重复选。
- 关键变量：`sum` 记录当前路径和；`startIndex` 控制枚举起点。
- 解题步骤：累加当前数字；如果超过目标就剪枝；如果正好等于目标就收集答案。
- 复杂度：和结果规模相关。
- 面试易错点：如果递归传 `i + 1`，就变成每个数只能用一次。
- 可复述话术：因为允许重复使用当前数字，所以递归层级里要保留当前下标。

### H_40 - 组合总和 II

- 题目定位：候选数组里有重复值，每个下标最多使用一次，结果不能重复。
- 当前代码思路：先排序，再用 `userd` 做同层去重。
- 关键变量：`userd[i - 1] == 0` 表示前一个相同值没有参与当前路径，是同层重复分支。
- 解题步骤：排序；从 `startIndex` 枚举；跳过同层重复；递归 `i + 1`；回溯恢复。
- 复杂度：和结果规模相关，排序 `O(n log n)`。
- 面试易错点：去重是“同层去重”，不是把所有重复值都跳掉。
- 可复述话术：排序后相同值相邻，只保留当前层第一次出现的那个分支。

### H_46 - 全排列

- 题目定位：返回数组的所有排列。
- 当前代码思路：每一层都从头扫描所有元素，用 `used[]` 记录已使用元素。
- 关键变量：`used[i]` 表示下标 i 是否已经进入当前路径。
- 解题步骤：跳过已使用元素；选择当前数；递归；撤销选择。
- 复杂度：时间 `O(n * n!)`，空间 `O(n)`。
- 面试易错点：排列题不能用 `startIndex` 限制只能向后选。
- 可复述话术：排列关注位置顺序，所以每一层都要从所有未使用元素里重新挑。

### H_47 - 全排列 II

- 题目定位：数组含重复数字，返回不重复的全排列。
- 当前代码思路：排序后在排列模板上增加同层去重条件。
- 关键变量：`!used[i - 1]` 表示前一个相同值没有在当前路径里，属于同层重复。
- 解题步骤：排序；枚举未使用元素；先跳过同层重复；再选择并递归。
- 复杂度：时间 `O(n * n!)`，空间 `O(n)`。
- 面试易错点：去重条件必须结合 `used[]` 一起判断。
- 可复述话术：先排序保证相同值相邻，再固定同层只选一次。

### H_51 - N 皇后

- 题目定位：在棋盘上放置 n 个皇后，要求互不攻击。
- 当前代码思路：按行回溯，每行尝试每一列，放之前检查列和对角线是否合法。
- 关键变量：`row` 表示当前行；`board` 保存棋盘状态。
- 解题步骤：逐行放置皇后；验证合法性；放置后递归下一行；回溯恢复。
- 复杂度：和有效解数量相关。
- 面试易错点：检查合法性时要同时看列、左上对角线和右上对角线。
- 可复述话术：我按行放皇后，每一行只放一个，放之前把冲突方向都检查一遍。

### H_131 - 分割回文串

- 题目定位：把字符串切成若干段，要求每一段都是回文。
- 当前代码思路：枚举每个切分点，只有当当前子串是回文时才继续向下递归。
- 关键变量：`start` 表示下一段切分起点；`path` 保存当前分割结果。
- 解题步骤：枚举 `end`；判断子串是否回文；加入路径；递归后续字符；撤销选择。
- 复杂度：和切分方案数量相关。
- 面试易错点：回文判断不要漏掉区间边界。
- 可复述话术：这是“切一刀再递归”的典型分割题，每段都要先验证回文再进入下一层。

### H_17 - 电话号码的字母组合

- 题目定位：每个数字对应一组字母，生成所有组合。
- 当前代码思路：按数字下标逐层递归，每层遍历当前数字对应的所有字母。
- 关键变量：`index` 指向当前数字；`path` 记录当前路径上的字符。
- 解题步骤：取当前数字；遍历对应字母；加入路径；递归下一位；撤销选择。
- 复杂度：时间 `O(3^m * 4^n)`，空间 `O(digits.length)`。
- 面试易错点：输入为空时通常要直接返回空集合。
- 可复述话术：这是典型的按位展开，每一层只负责当前数字能选哪些字母。

### H_22 - 括号生成

- 题目定位：生成所有合法的 n 对括号。
- 当前代码思路：记录左右括号数量，只允许前缀保持合法。
- 关键变量：`left/right` 表示已放入的括号数量。
- 解题步骤：左括号没用完就可以继续放；右括号必须少于左括号才可以放；长度到 `2 * n` 时收集。
- 复杂度：答案数量是卡特兰数，空间 `O(n)`。
- 面试易错点：右括号不能超过左括号，否则前缀已经非法。
- 可复述话术：我用“前缀合法”作为剪枝条件，所有非法路径都在递归过程中直接舍弃。

### H_491 - 递增子序列

- 题目定位：找长度至少为 2 的非递减子序列。
- 当前代码思路：不能排序，因此每层递归用 `Set` 去重，同时保证路径非递减。
- 关键变量：`path` 保存当前子序列；`uset` 记录当前层已经使用过的值。
- 解题步骤：如果路径长度大于 1 就收集；本层新建 `Set`；遍历后续元素；过滤下降和同层重复；递归回溯。
- 复杂度：时间 `O(n * 2^n)`，空间 `O(n)`。
- 面试易错点：不能排序，否则会破坏原数组顺序，子序列就不成立了。
- 可复述话术：这题不能靠排序去重，所以必须在每一层单独记录本层用过的值。

### H_79 - 单词搜索

- 题目定位：在二维字符网格中寻找一个单词，路径必须按单词字符顺序连接，每一步只能向上下左右移动，同一个格子在一条路径中不能重复使用。
- 当前代码思路：先遍历整个网格，找到所有等于 `word.charAt(0)` 的起点；从每个起点调用 `dfs`，递归匹配当前字符和后续字符。
- 递归含义：`dfs(board, word, x, y, index)` 表示尝试从坐标 `(x,y)` 开始，匹配 `word` 的第 `index` 个字符以及后续部分。
- 关键变量：`x/y` 是当前坐标；`index` 是当前要匹配的单词下标；`chara` 保存当前格子的原字符，用于递归结束后恢复现场。
- 解题步骤：先判断坐标是否越界；再判断当前格子是否等于 `word.charAt(index)`；如果已经匹配到最后一个字符，直接返回 `true`；否则把当前格子临时改成 `'#'`，继续向四个方向搜索下一个字符，最后恢复原字符。
- 回溯细节：`board[x][y] = '#'` 的作用等价于 visited 标记，防止当前路径重复使用同一个格子；递归结束后必须恢复为 `chara`，否则会影响其他路径和其他起点。
- 剪枝点：外层只有首字母匹配才进入 DFS；DFS 内部越界或字符不匹配立即返回 `false`。
- 复杂度：设网格为 `m * n`，单词长度为 `L`，最坏时间 `O(m * n * 4^L)`；递归栈空间 `O(L)`。
- 面试易错点：最后一个字符匹配成功就可以返回，不需要继续递归；临时修改网格后一定要恢复；边界判断必须放在访问 `board[x][y]` 之前。
- 可复述话术：我先枚举每个可能的首字母起点，然后 DFS 按顺序匹配单词；每走过一个格子就临时标记，四个方向搜索完再恢复，这样既避免重复使用格子，也不影响其他路径。

## 3. 通用解题模板

### 组合模板

```java
List<List<Integer>> result = new ArrayList<>();
List<Integer> path = new ArrayList<>();

void back(int[] nums, int startIndex) {
    if (满足收集条件) {
        result.add(new ArrayList<>(path));
        return;
    }

    for (int i = startIndex; i < nums.length; i++) {
        path.add(nums[i]);
        back(nums, i + 1);
        path.removeLast();
    }
}
```

### 可重复选择组合模板

`H_39.java` 的关键是递归时继续传 `i`，表示当前元素还能继续被选。

```java
for (int i = startIndex; i < candidates.length; i++) {
    path.add(candidates[i]);
    sum += candidates[i];
    backtracking(candidates, target, sum, i);
    sum -= candidates[i];
    path.removeLast();
}
```

### 不可重复选择组合模板

`H_40.java` 的关键是递归时传 `i + 1`，表示每个下标最多使用一次。

```java
for (int i = startIndex; i < candidates.length; i++) {
    path.add(candidates[i]);
    sum += candidates[i];
    backtrack(candidates, target, sum, i + 1, userd);
    path.removeLast();
    sum -= candidates[i];
}
```

### 组合剪枝模板

`H_77_剪枝.java` 中如果还需要 `k - path.size()` 个数，那么 `i` 最多只能到：

```java
for (int i = startIndex; i <= n - (k - path.size()) + 1; i++) {
    path.add(i);
    back(n, k, i + 1);
    path.removeLast();
}
```

### 排列模板

```java
void dfs(int[] nums, boolean[] used) {
    if (path.size() == nums.length) {
        result.add(new ArrayList<>(path));
        return;
    }

    for (int i = 0; i < nums.length; i++) {
        if (used[i]) {
            continue;
        }
        used[i] = true;
        path.add(nums[i]);
        dfs(nums, used);
        used[i] = false;
        path.removeLast();
    }
}
```

### 排列去重模板

`H_47.java` 先排序，再跳过同一层中前一个相同值还没被使用的情况。

```java
if (i > 0 && nums[i] == nums[i - 1] && !used[i - 1]) {
    continue;
}
```

### 子序列去重模板

`H_491.java` 不能排序，因为要保留原数组相对顺序，所以每一层新建 `Set` 记录本层用过的值。

```java
Set<Integer> uset = new HashSet<>();
for (int i = startIndex; i < nums.length; i++) {
    if ((!path.isEmpty() && nums[i] < path.getLast()) || uset.contains(nums[i])) {
        continue;
    }
    path.add(nums[i]);
    uset.add(nums[i]);
    dfs(nums, i + 1);
    path.removeLast();
}
```

## 5. 每题详解

### H_77 - 组合

- 题目定位：从 `1..n` 中选择 `k` 个数。
- 当前代码思路：全局 `result/path`，递归参数 `startIndex` 表示本层从哪个数字开始枚举。
- 关键变量：`path.size() == k` 时收集答案；递归下一层传 `i + 1`。
- 解题步骤：枚举当前层可选数字；加入 `path`；递归选后续数字；撤销选择。
- 复杂度：时间 `O(k * C(n,k))`，空间 `O(k)`。
- 面试易错点：不能每层都从 `1` 开始，否则会出现重复组合和死循环。
- 可复述话术：组合不关心顺序，所以用 `startIndex` 保证后面只能选比当前下标更靠后的数。

### H_77_剪枝 - 组合，剪枝版

- 题目定位：`H_77` 的优化版本。
- 当前代码思路：如果剩余元素不足以填满 `k` 个数，循环提前结束。
- 关键变量：`n - (k - path.size()) + 1` 是当前层 `i` 的最大可选值。
- 解题步骤：先判断是否已选够；再按剪枝边界枚举；递归和回溯。
- 复杂度：理论结果规模不变，但减少无效分支。
- 面试易错点：右边界要加 `1`，因为数字范围是闭区间。
- 可复述话术：当前还差几个数，就必须给后续保留几个位置，所以循环不用走到 `n`。

### H_216 - 组合总和 III

- 题目定位：从 `1..9` 中选 `k` 个数，使和为 `n`。
- 当前代码思路：`startIndex` 控制不可重复选择，`sum` 记录当前总和。
- 关键变量：`sum > n` 直接返回；`path.size() == k && sum == n` 收集答案。
- 解题步骤：从 `startIndex` 到 `9` 枚举；加入数字并累加；递归 `i + 1`；回溯还原。
- 复杂度：时间和有效组合数量相关，空间 `O(k)`。
- 面试易错点：必须同时满足数量和总和，不能只看 `sum == n`。
- 可复述话术：这是固定候选范围的组合题，用 `startIndex` 控制每个数字只用一次，用 `sum` 做提前剪枝。

### H_39 - 组合总和

- 题目定位：候选数字可以重复使用，求和为 `target` 的组合。
- 当前代码思路：递归传 `i`，让当前候选数下一层还能继续被选。
- 关键变量：`sum > target` 返回；`sum == target` 收集答案。
- 解题步骤：从 `startIndex` 枚举候选数；加入并累加；递归传 `i`；回溯还原 `sum` 和 `path`。
- 复杂度：和目标值、候选数、答案数量相关；递归空间和路径长度相关。
- 面试易错点：如果递归传 `i + 1`，就变成每个元素只能用一次。
- 可复述话术：因为元素可以重复使用，所以选择当前数后，下一层仍然从当前下标开始搜索。

### H_40 - 组合总和 II

- 题目定位：候选数组可能有重复值，但每个下标最多使用一次，答案不能重复。
- 当前代码思路：先排序，再用 `userd` 判断同层重复值是否要跳过。
- 关键变量：`i > 0 && candidates[i] == candidates[i - 1] && userd[i - 1] == 0` 表示同层重复。
- 解题步骤：排序；从 `startIndex` 枚举；跳过同层重复；递归传 `i + 1`；回溯恢复。
- 复杂度：和结果规模相关，排序 `O(n log n)`，递归空间 `O(n)`。
- 面试易错点：去重不是简单跳过所有重复值，而是跳过同一层中重复的选择。
- 可复述话术：排序后相同值相邻，当前一个相同值没有在当前路径中使用时，说明这是同层重复分支，要跳过。

### H_46 - 全排列

- 题目定位：返回数组的所有排列。
- 当前代码思路：每一层都从头枚举数组，用 `used[]` 排除当前路径已使用元素。
- 关键变量：`path.size() == nums.length` 收集答案。
- 解题步骤：枚举所有下标；跳过 `used[i]`；选择；递归；撤销 `used` 和 `path`。
- 复杂度：时间 `O(n * n!)`，空间 `O(n)`。
- 面试易错点：排列关注顺序，所以不能用 `startIndex` 限制只能向后选。
- 可复述话术：排列每个位置都可能放任意未使用元素，所以用 `used[]` 表示哪些数已经进入当前路径。

### H_47 - 全排列 II

- 题目定位：数组包含重复数字，返回不重复的全排列。
- 当前代码思路：排序后在全排列模板上增加同层去重条件。
- 关键变量：`i > 0 && nums[i] == nums[i - 1] && !used[i - 1]`。
- 解题步骤：排序；枚举下标；先跳过同层重复；再跳过已使用元素；递归回溯。
- 复杂度：时间 `O(n * n!)`，空间 `O(n)`。
- 面试易错点：去重条件中的 `!used[i - 1]` 表示前一个相同值没有在当前路径里，属于同层重复。
- 可复述话术：重复元素要固定相对选择顺序，避免同一层用相同值开启两个等价分支。

### H_491 - 递增子序列

- 题目定位：找长度至少为 2 的非递减子序列。
- 当前代码思路：保留原数组顺序，不能排序；每层用 `HashSet` 去重。
- 关键变量：`path.size() > 1` 立即收集；`nums[i] < path.getLast()` 跳过下降元素。
- 解题步骤：若当前路径长度大于 1 就收集；本层创建 `uset`；枚举后续元素；过滤下降和本层重复；递归。
- 复杂度：时间 `O(n * 2^n)`，空间 `O(n)`。
- 面试易错点：不能排序去重，否则会破坏子序列的原始顺序。
- 可复述话术：子序列必须保持原数组顺序，因此用每层 `Set` 去重，而不是排序后去重。

### H_17 - 电话号码的字母组合

- 题目定位：每个数字对应一组字母，求所有字符串组合。
- 当前代码思路：`index` 表示处理到第几个数字，每层枚举该数字对应的所有字母。
- 关键变量：`phoneMap` 映射数字到字母；`index == digits.length()` 收集字符串。
- 解题步骤：取当前数字；遍历对应字母；加入 `path`；递归处理下一个数字；撤销。
- 复杂度：时间 `O(3^m * 4^n)`，空间 `O(digits.length)`。
- 面试易错点：空字符串输入时通常应直接返回空列表；当前代码会在 `digits` 为空时收集一个空字符串，面试中要按题目要求处理。
- 可复述话术：这是按位构造字符串，每一层只负责当前数字能选哪个字母。

### H_22 - 括号生成

- 题目定位：生成所有合法的 `n` 对括号。
- 当前代码思路：用 `left/right` 记录已经放入的左右括号数量。
- 关键变量：`left < n` 时可以放左括号；`right < left` 时可以放右括号。
- 解题步骤：路径长度达到 `2 * n` 收集；尝试放左括号；尝试放右括号；分别回溯。
- 复杂度：答案数量是卡特兰数，空间 `O(n)`。
- 面试易错点：右括号数量不能超过左括号，否则前缀已经非法。
- 可复述话术：合法括号的每个前缀都必须满足右括号数不超过左括号数，所以递归时用这个约束剪枝。

### H_79 - 单词搜索

- 题目定位：判断二维字符网格中是否存在一条路径，能够按顺序组成目标单词。
- 当前代码思路：外层双重循环枚举所有格子，只有格子字符等于单词首字母时才调用 `dfs`；DFS 每次负责匹配 `word[index]`，匹配成功后继续向四个方向查找下一个字符。
- 递归含义：`dfs(board, word, x, y, index)` 表示当前在 `(x,y)` 这个格子，尝试匹配 `word[index]`，并判断从这里能否继续匹配完整个单词。
- 终止条件：坐标越界返回 `false`；当前格子和 `word[index]` 不相等返回 `false`；如果 `index == word.length() - 1` 且当前字符已经匹配，说明单词已经全部找到，返回 `true`。
- 回溯处理：用 `chara` 保存当前字符，然后把 `board[x][y]` 改成 `'#'` 表示当前路径已访问；四个方向搜索结束后恢复 `board[x][y] = chara`。
- 四个方向：当前代码按下、上、右、左的顺序搜索，四个递归调用之间用 `||` 连接，只要一个方向成功就整体成功。
- 复杂度：时间最坏 `O(m * n * 4^L)`，其中 `L` 是单词长度；空间 `O(L)`，来自递归深度。
- 面试易错点：不能在一条路径里重复使用同一个格子；如果用原地标记代替 `visited`，必须在返回前恢复；边界判断要先于字符访问。
- 可复述话术：我把每个首字母匹配的位置当作起点做 DFS，每一步匹配当前字符并临时标记已访问，继续向四个方向找下一个字符，递归结束后恢复现场。

## 6. 面试高频易错点

- 收集答案时必须 `new ArrayList<>(path)`，不能直接放 `path` 引用。
- 组合问题递归下一层一般传 `i + 1`，可重复选择才传 `i`。
- 排列问题不能用 `startIndex`，因为每个位置都要从全部未使用元素里选。
- 去重前通常要排序，但递增子序列这类保持原顺序的题不能排序。
- 回溯后要恢复所有被修改的状态，例如 `path`、`sum`、`used`。
- 剪枝条件要保证不漏解，只能剪掉确定不可能产生答案的分支。
