## 2. 本专题核心思想

- 回溯就是在递归树上做选择、进入下一层、撤销选择。
- `path` 表示当前路径，`result` 表示所有合法答案。
- 组合问题靠 `startIndex` 防止重复选择之前的元素。
- 排列问题没有 `startIndex`，靠 `used[]` 判断当前路径是否已经使用过某个元素。
- 去重先想清楚是树层去重还是树枝去重：同一层不要选重复值，路径内部可以根据题意决定是否允许重复。
- 剪枝的本质是提前停止不可能产生答案的分支。

## 3. 通用解题模板

### 组合模板

```java
List<List<Integer>> result = new ArrayList<>();
List<Integer> path = new ArrayList<>();

void back(int[] nums, int startIndex) {
    if (满足收集条件) {
        result.add(new ArrayList<>(path));
        return;
    }

    for (int i = startIndex; i < nums.length; i++) {
        path.add(nums[i]);
        back(nums, i + 1);
        path.removeLast();
    }
}
```

### 可重复选择组合模板

`H_39.java` 的关键是递归时继续传 `i`，表示当前元素还能继续被选。

```java
for (int i = startIndex; i < candidates.length; i++) {
    path.add(candidates[i]);
    sum += candidates[i];
    backtracking(candidates, target, sum, i);
    sum -= candidates[i];
    path.removeLast();
}
```

### 不可重复选择组合模板

`H_40.java` 的关键是递归时传 `i + 1`，表示每个下标最多使用一次。

```java
for (int i = startIndex; i < candidates.length; i++) {
    path.add(candidates[i]);
    sum += candidates[i];
    backtrack(candidates, target, sum, i + 1, userd);
    path.removeLast();
    sum -= candidates[i];
}
```

### 组合剪枝模板

`H_77_剪枝.java` 中如果还需要 `k - path.size()` 个数，那么 `i` 最多只能到：

```java
for (int i = startIndex; i <= n - (k - path.size()) + 1; i++) {
    path.add(i);
    back(n, k, i + 1);
    path.removeLast();
}
```

### 排列模板

```java
void dfs(int[] nums, boolean[] used) {
    if (path.size() == nums.length) {
        result.add(new ArrayList<>(path));
        return;
    }

    for (int i = 0; i < nums.length; i++) {
        if (used[i]) {
            continue;
        }
        used[i] = true;
        path.add(nums[i]);
        dfs(nums, used);
        used[i] = false;
        path.removeLast();
    }
}
```

### 排列去重模板

`H_47.java` 先排序，再跳过同一层中前一个相同值还没被使用的情况。

```java
if (i > 0 && nums[i] == nums[i - 1] && !used[i - 1]) {
    continue;
}
```

### 子序列去重模板

`H_491.java` 不能排序，因为要保留原数组相对顺序，所以每一层新建 `Set` 记录本层用过的值。

```java
Set<Integer> uset = new HashSet<>();
for (int i = startIndex; i < nums.length; i++) {
    if ((!path.isEmpty() && nums[i] < path.getLast()) || uset.contains(nums[i])) {
        continue;
    }
    path.add(nums[i]);
    uset.add(nums[i]);
    dfs(nums, i + 1);
    path.removeLast();
}
```

## 5. 每题详解

### H_77 - 组合

- 题目定位：从 `1..n` 中选择 `k` 个数。
- 当前代码思路：全局 `result/path`，递归参数 `startIndex` 表示本层从哪个数字开始枚举。
- 关键变量：`path.size() == k` 时收集答案；递归下一层传 `i + 1`。
- 解题步骤：枚举当前层可选数字；加入 `path`；递归选后续数字；撤销选择。
- 复杂度：时间 `O(k * C(n,k))`，空间 `O(k)`。
- 面试易错点：不能每层都从 `1` 开始，否则会出现重复组合和死循环。
- 可复述话术：组合不关心顺序，所以用 `startIndex` 保证后面只能选比当前下标更靠后的数。

### H_77_剪枝 - 组合，剪枝版

- 题目定位：`H_77` 的优化版本。
- 当前代码思路：如果剩余元素不足以填满 `k` 个数，循环提前结束。
- 关键变量：`n - (k - path.size()) + 1` 是当前层 `i` 的最大可选值。
- 解题步骤：先判断是否已选够；再按剪枝边界枚举；递归和回溯。
- 复杂度：理论结果规模不变，但减少无效分支。
- 面试易错点：右边界要加 `1`，因为数字范围是闭区间。
- 可复述话术：当前还差几个数，就必须给后续保留几个位置，所以循环不用走到 `n`。

### H_216 - 组合总和 III

- 题目定位：从 `1..9` 中选 `k` 个数，使和为 `n`。
- 当前代码思路：`startIndex` 控制不可重复选择，`sum` 记录当前总和。
- 关键变量：`sum > n` 直接返回；`path.size() == k && sum == n` 收集答案。
- 解题步骤：从 `startIndex` 到 `9` 枚举；加入数字并累加；递归 `i + 1`；回溯还原。
- 复杂度：时间和有效组合数量相关，空间 `O(k)`。
- 面试易错点：必须同时满足数量和总和，不能只看 `sum == n`。
- 可复述话术：这是固定候选范围的组合题，用 `startIndex` 控制每个数字只用一次，用 `sum` 做提前剪枝。

### H_39 - 组合总和

- 题目定位：候选数字可以重复使用，求和为 `target` 的组合。
- 当前代码思路：递归传 `i`，让当前候选数下一层还能继续被选。
- 关键变量：`sum > target` 返回；`sum == target` 收集答案。
- 解题步骤：从 `startIndex` 枚举候选数；加入并累加；递归传 `i`；回溯还原 `sum` 和 `path`。
- 复杂度：和目标值、候选数、答案数量相关；递归空间和路径长度相关。
- 面试易错点：如果递归传 `i + 1`，就变成每个元素只能用一次。
- 可复述话术：因为元素可以重复使用，所以选择当前数后，下一层仍然从当前下标开始搜索。

### H_40 - 组合总和 II

- 题目定位：候选数组可能有重复值，但每个下标最多使用一次，答案不能重复。
- 当前代码思路：先排序，再用 `userd` 判断同层重复值是否要跳过。
- 关键变量：`i > 0 && candidates[i] == candidates[i - 1] && userd[i - 1] == 0` 表示同层重复。
- 解题步骤：排序；从 `startIndex` 枚举；跳过同层重复；递归传 `i + 1`；回溯恢复。
- 复杂度：和结果规模相关，排序 `O(n log n)`，递归空间 `O(n)`。
- 面试易错点：去重不是简单跳过所有重复值，而是跳过同一层中重复的选择。
- 可复述话术：排序后相同值相邻，当前一个相同值没有在当前路径中使用时，说明这是同层重复分支，要跳过。

### H_46 - 全排列

- 题目定位：返回数组的所有排列。
- 当前代码思路：每一层都从头枚举数组，用 `used[]` 排除当前路径已使用元素。
- 关键变量：`path.size() == nums.length` 收集答案。
- 解题步骤：枚举所有下标；跳过 `used[i]`；选择；递归；撤销 `used` 和 `path`。
- 复杂度：时间 `O(n * n!)`，空间 `O(n)`。
- 面试易错点：排列关注顺序，所以不能用 `startIndex` 限制只能向后选。
- 可复述话术：排列每个位置都可能放任意未使用元素，所以用 `used[]` 表示哪些数已经进入当前路径。

### H_47 - 全排列 II

- 题目定位：数组包含重复数字，返回不重复的全排列。
- 当前代码思路：排序后在全排列模板上增加同层去重条件。
- 关键变量：`i > 0 && nums[i] == nums[i - 1] && !used[i - 1]`。
- 解题步骤：排序；枚举下标；先跳过同层重复；再跳过已使用元素；递归回溯。
- 复杂度：时间 `O(n * n!)`，空间 `O(n)`。
- 面试易错点：去重条件中的 `!used[i - 1]` 表示前一个相同值没有在当前路径里，属于同层重复。
- 可复述话术：重复元素要固定相对选择顺序，避免同一层用相同值开启两个等价分支。

### H_491 - 递增子序列

- 题目定位：找长度至少为 2 的非递减子序列。
- 当前代码思路：保留原数组顺序，不能排序；每层用 `HashSet` 去重。
- 关键变量：`path.size() > 1` 立即收集；`nums[i] < path.getLast()` 跳过下降元素。
- 解题步骤：若当前路径长度大于 1 就收集；本层创建 `uset`；枚举后续元素；过滤下降和本层重复；递归。
- 复杂度：时间 `O(n * 2^n)`，空间 `O(n)`。
- 面试易错点：不能排序去重，否则会破坏子序列的原始顺序。
- 可复述话术：子序列必须保持原数组顺序，因此用每层 `Set` 去重，而不是排序后去重。

### H_17 - 电话号码的字母组合

- 题目定位：每个数字对应一组字母，求所有字符串组合。
- 当前代码思路：`index` 表示处理到第几个数字，每层枚举该数字对应的所有字母。
- 关键变量：`phoneMap` 映射数字到字母；`index == digits.length()` 收集字符串。
- 解题步骤：取当前数字；遍历对应字母；加入 `path`；递归处理下一个数字；撤销。
- 复杂度：时间 `O(3^m * 4^n)`，空间 `O(digits.length)`。
- 面试易错点：空字符串输入时通常应直接返回空列表；当前代码会在 `digits` 为空时收集一个空字符串，面试中要按题目要求处理。
- 可复述话术：这是按位构造字符串，每一层只负责当前数字能选哪个字母。

### H_22 - 括号生成

- 题目定位：生成所有合法的 `n` 对括号。
- 当前代码思路：用 `left/right` 记录已经放入的左右括号数量。
- 关键变量：`left < n` 时可以放左括号；`right < left` 时可以放右括号。
- 解题步骤：路径长度达到 `2 * n` 收集；尝试放左括号；尝试放右括号；分别回溯。
- 复杂度：答案数量是卡特兰数，空间 `O(n)`。
- 面试易错点：右括号数量不能超过左括号，否则前缀已经非法。
- 可复述话术：合法括号的每个前缀都必须满足右括号数不超过左括号数，所以递归时用这个约束剪枝。

## 6. 面试高频易错点

- 收集答案时必须 `new ArrayList<>(path)`，不能直接放 `path` 引用。
- 组合问题递归下一层一般传 `i + 1`，可重复选择才传 `i`。
- 排列问题不能用 `startIndex`，因为每个位置都要从全部未使用元素里选。
- 去重前通常要排序，但递增子序列这类保持原顺序的题不能排序。
- 回溯后要恢复所有被修改的状态，例如 `path`、`sum`、`used`。
- 剪枝条件要保证不漏解，只能剪掉确定不可能产生答案的分支。
