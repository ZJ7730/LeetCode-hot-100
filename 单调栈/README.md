## 4. 易错点

- 栈里通常存下标，不只存值，因为需要计算距离或宽度。
- 明确维护的是递增栈还是递减栈。
- `84` 推荐在数组两端加哨兵高度 `0`，简化清栈逻辑。
- `42` 双指针解法中 `leftMax`、`rightMax` 由 `Math.max` 维护，不能手动 `++`，否则会人为抬高短板。

## 5. 每题详解

### H_42 - 接雨水

- 题目定位：给定每个位置的柱子高度，求下雨后总共能接多少雨水。
- 当前代码思路：双指针对撞 + 维护两侧的历史最大高度，每一步在短板侧结算雨水。
- 核心公式：位置 `i` 能接的雨水 = `min(leftMax, rightMax) - height[i]`，谁是短板谁决定雨水量。
- 关键变量：
  - `left`、`right`：左右指针，从两端往中间逼近。
  - `leftMax`：`left` 左侧（含）的历史最大高度。
  - `rightMax`：`right` 右侧（含）的历史最大高度。
  - `res`：累计的雨水总量。
- 解题步骤：
  1. 初始化 `left = 0, right = n - 1`，`leftMax = rightMax = 0`。
  2. 每次循环先同步更新 `leftMax = max(leftMax, height[left])`、`rightMax = max(rightMax, height[right])`。
  3. 若 `leftMax <= rightMax`，对 `left` 位置而言短板是 `leftMax`（因为右侧已知有一个 >= leftMax 的高度），结算 `leftMax - height[left]`，`left++`。
  4. 否则短板是 `rightMax`，结算 `rightMax - height[right]`，`right--`。
  5. 直到 `left > right` 结束。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：
  - `leftMax` 和 `rightMax` 是历史最大值，**必须用 `Math.max` 维护，不能写成 `leftMax++`**，否则结算时差值会被人为放大。
  - 短板一侧才能结算，`leftMax <= rightMax` 时一定移动 `left`，反之移动 `right`。
- 可复述话术：每个位置能接多少雨水由两侧最高柱子中较矮的那个决定。我用左右指针对撞，每次只在短板那一侧结算雨水，因为短板侧不需要知道远处具体地形——只要远处存在一个 >= 当前侧最大高度的柱子就够了。
