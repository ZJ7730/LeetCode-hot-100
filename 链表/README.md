## 2. 本专题核心思想

- 链表题先想是否需要 `dummy`，只要可能删除或改头节点，优先加虚拟头节点。
- 指针变更前先保存后继节点，典型写法是 `ListNode tmp = cur.next`。
- 快慢指针适合找中点、倒数第 N 个、判断环和找环入口。
- 反转链表是链表题的基础模板，区间反转、K 个一组反转、回文链表都会复用。
- 合并和构造新链表时，用 `tail` 指向结果链表尾部。
- 链表面试最容易错在空指针、头节点变化、循环边界。

## 3. 通用解题模板

### ListNode 结构

当前目录的公共节点定义是：

```java
public class ListNode {
    int val;
    ListNode next;
}
```

LeetCode 面试中通常已经提供 `ListNode`，重点是指针操作。

### 虚拟头节点模板

```java
ListNode dummy = new ListNode(-1);
dummy.next = head;
ListNode pre = dummy;

// 修改链表

return dummy.next;
```

使用场景：删除节点、合并链表、反转区间、可能改变头节点的题。

### 反转链表迭代模板

对应 `H_206.java`：

```java
ListNode cur = head;
ListNode pre = null;
while (cur != null) {
    ListNode tmp = cur.next;
    cur.next = pre;
    pre = cur;
    cur = tmp;
}
return pre;
```

### 反转链表递归模板

对应 `H_206.java` 的 `reverseList2`：

```java
ListNode dfs(ListNode cur, ListNode pre) {
    if (cur == null) {
        return pre;
    }
    ListNode tmp = cur.next;
    cur.next = pre;
    return dfs(tmp, cur);
}
```

### 快慢指针判环模板

```java
ListNode fast = head;
ListNode slow = head;
while (fast != null && fast.next != null) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow == fast) {
        return true;
    }
}
return false;
```

### 删除倒数第 N 个节点模板

对应 `H_19.java`：

```java
ListNode dummy = new ListNode(-1);
dummy.next = head;
ListNode fast = dummy;
ListNode slow = dummy;

for (int i = 0; i <= n; i++) {
    fast = fast.next;
}
while (fast != null) {
    fast = fast.next;
    slow = slow.next;
}
slow.next = slow.next.next;
return dummy.next;
```

### 区间反转头插法模板

对应 `H_92.java`：

```java
ListNode dummy = new ListNode(-1);
dummy.next = head;
ListNode pre = dummy;
ListNode tail = head;

for (int i = 1; i < left; i++) {
    pre = pre.next;
    tail = tail.next;
}

for (int i = left; i < right; i++) {
    ListNode move = tail.next;
    tail.next = move.next;
    move.next = pre.next;
    pre.next = move;
}
return dummy.next;
```

## 5. 每题详解

### H_21 - 合并两个有序链表

- 题目定位：两个升序链表合并成一个升序链表。
- 当前代码思路：使用 `dummy` 和 `tail`，每次把较小节点接到结果链表后面。
- 关键变量：`list1/list2` 是两个当前比较节点；`tail` 是结果链表尾部。
- 解题步骤：比较两个节点值；接较小节点；移动对应链表和 `tail`；最后接上剩余链表。
- 复杂度：时间 `O(m+n)`，空间 `O(1)`。
- 面试易错点：循环结束后要把未遍历完的一条链表整体接到 `tail.next`。
- 可复述话术：这就是链表版归并，用 dummy 简化头节点处理，用 tail 维护结果链表末尾。

### H_206 - 反转链表

- 题目定位：反转整个单链表。
- 当前代码思路：迭代写法用 `cur/pre/tmp` 三个指针；递归写法把当前节点指向前驱。
- 关键变量：`tmp` 保存原来的下一个节点，防止断链。
- 解题步骤：保存 `cur.next`；让 `cur.next = pre`；`pre/cur` 同步后移；最后返回 `pre`。
- 复杂度：迭代时间 `O(n)`、空间 `O(1)`；递归空间 `O(n)`。
- 面试易错点：一定要先保存 `tmp`，否则改 `cur.next` 后找不到后续链表。
- 可复述话术：每次把当前节点的 next 指向前一个节点，然后三个指针整体往后移动。

### H_2 - 两数相加

- 题目定位：两个逆序链表表示整数，返回相加后的逆序链表。
- 当前代码思路：`dummy + tail` 构造结果链表，用 `carry` 记录进位。
- 关键变量：`x/y` 处理某个链表为空时补 `0`；`sum % 10` 是当前位；`sum / 10` 是进位。
- 解题步骤：只要 `l1/l2/carry` 还有值就继续；计算当前位；新建节点接到 `tail`；移动输入链表。
- 复杂度：时间 `O(max(m,n))`，空间 `O(max(m,n))`。
- 面试易错点：循环条件要包含 `carry != 0`，否则最后一位进位会丢失。
- 可复述话术：按位模拟加法，两个链表长度不同就把缺失位当作 0，进位继续参与下一轮。

### H_19 - 删除倒数第 N 个节点

- 题目定位：删除链表倒数第 `n` 个节点。
- 当前代码思路：`dummy` 统一处理删除头节点；`fast` 比 `slow` 先走 `n + 1` 步。
- 关键变量：当 `fast == null` 时，`slow` 正好停在待删除节点的前一个节点。
- 解题步骤：建立 dummy；fast 先走；fast/slow 同步走；修改 `slow.next` 删除节点。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：从 `dummy` 出发时 fast 走 `n + 1` 步，保证 slow 是删除节点前驱。
- 可复述话术：让快慢指针保持固定间距，快指针到尾时慢指针就在删除位置前一个节点。

### H_141 - 环形链表

- 题目定位：判断链表是否有环。
- 当前代码思路：快慢指针，快指针每次两步，慢指针每次一步。
- 关键变量：`fast != null && fast.next != null` 保证 `fast.next.next` 安全。
- 解题步骤：循环移动快慢指针；如果相遇说明有环；否则快指针到空说明无环。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：循环条件不能只判断 `fast != null`。
- 可复述话术：如果有环，快指针一定会在环里追上慢指针；如果无环，快指针会先到 null。

### H_142 - 环形链表 II

- 题目定位：返回链表入环的第一个节点。
- 当前代码思路：先用快慢指针找到相遇点 `end`，再让 `start` 从头出发，`end` 从相遇点出发，同步走到入口。
- 关键变量：`end == null` 表示无环；`start == end` 时就是入口。
- 解题步骤：快慢指针找相遇；如果没有相遇返回 null；头指针和相遇点指针同步走；返回入口。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：找入口时两个指针都每次走一步，不是快慢走。
- 可复述话术：根据环形链表的距离关系，从头节点和相遇点同时出发，会在环入口第一次相遇。

### H_160 - 相交链表

- 题目定位：找两个链表的第一个公共节点。
- 当前代码思路：`pA` 走完 A 后切到 B，`pB` 走完 B 后切到 A，使两个指针总路程相同。
- 关键变量：`while (pA != pB)`，相等时可能是交点，也可能都是 null。
- 解题步骤：两个指针同步走；为空则切换到另一条链表头；最终相遇返回。
- 复杂度：时间 `O(m+n)`，空间 `O(1)`。
- 面试易错点：比较的是节点引用是否相同，不是节点值是否相同。
- 可复述话术：两个指针都走 A+B 的总长度，长度差被切换链表抵消，所以会在交点同步相遇。

### H_234 - 回文链表

- 题目定位：判断链表正读反读是否一致。
- 当前代码思路：快慢指针找到中点，反转后半段，再从头和后半段开头逐个比较。
- 关键变量：`slow` 最终指向后半段起点；`reverse` 是反转后的后半段头节点。
- 解题步骤：找中点；反转 slow 开始的链表；双指针比较前半段和反转后半段。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：奇数长度时后半段包含中间节点也不影响比较，因为只比较 `cur1 != null` 的长度。
- 可复述话术：回文链表可以转成前半段和反转后的后半段是否相同的问题。

### H_92 - 反转链表 II

- 题目定位：只反转链表第 `left` 到 `right` 个节点。
- 当前代码思路：先定位 `pre` 和 `tail`，再用头插法把区间内后面的节点不断移到 `pre` 后面。
- 关键变量：`pre` 是反转区间前一个节点；`tail` 是反转后区间尾节点；`move` 是每次被移动到前面的节点。
- 解题步骤：dummy 连接 head；移动到 left 前；循环 `right - left` 次头插；返回 dummy.next。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：`tail` 在整个头插过程中不动，它始终是反转区间的尾部。
- 可复述话术：区间反转可以不用整体断开链表，每次把 tail 后面的节点摘下来插到 pre 后面。

## 6. 面试高频易错点

- 删除或反转可能涉及头节点时，优先使用 `dummy`。
- 修改 `next` 前一定先保存后续节点。
- 快慢指针循环里要判断 `fast != null && fast.next != null`。
- 比较链表相交要比较节点引用，不是值。
- 区间反转中要分清 `pre`、`tail`、`move` 的含义。
- 回文链表如果要求恢复原链表，比较后还需要把后半段再反转回来；当前代码没有恢复，LeetCode 通常允许。
