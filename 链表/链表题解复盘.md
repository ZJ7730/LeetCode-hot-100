## H_141 - 环形链表

### 核心思路

使用快慢指针判断链表是否有环：

- `slow` 每次走 1 步。
- `fast` 每次走 2 步。
- 如果链表有环，`fast` 会在环里追上 `slow`。
- 如果链表无环，`fast` 会先走到 `null`。

### 题解代码

```java
public boolean hasCycle(ListNode head) {
    ListNode fast = head;
    ListNode slow = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;

        if (slow == fast) {
            return true;
        }
    }

    return false;
}
```

### 快速理解

无环链表：

```text
1 -> 2 -> 3 -> null
```

`fast` 每次走两步，最终一定会遇到 `null`，说明没有环。

有环链表：

```text
3 -> 2 -> 0 -> -4
     ^         |
     |_________|
```

进入环后，`fast` 每轮比 `slow` 多走 1 步，所以一定会追上 `slow`。

### 易错点

- 判断相遇要用 `slow == fast`，比较的是同一个节点引用，不是节点值。
- 循环条件必须是 `fast != null && fast.next != null`，否则 `fast.next.next` 可能空指针。

### 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## H_21 - 合并两个有序链表

### 核心思路

使用 `dummy + tail` 合并两个升序链表：

- `dummy` 是虚拟头节点，简化头节点处理。
- `tail` 指向结果链表的尾部。
- 每次比较 `list1.val` 和 `list2.val`，把较小节点接到 `tail.next`。
- 某一个链表走完后，把另一个链表剩余部分直接接上。

### 题解代码

```java
public ListNode mergeTwoLists(ListNode list1, ListNode list2) {
    ListNode dummy = new ListNode(-1);
    ListNode tail = dummy;

    while (list1 != null && list2 != null) {
        if (list1.val < list2.val) {
            tail.next = list1;
            list1 = list1.next;
        } else {
            tail.next = list2;
            list2 = list2.next;
        }
        tail = tail.next;
    }

    if (list1 != null) {
        tail.next = list1;
    }
    if (list2 != null) {
        tail.next = list2;
    }

    return dummy.next;
}
```

### 快速理解

示例：

```text
list1: 1 -> 2 -> 4
list2: 1 -> 3 -> 4
```

每次取两个链表当前节点中较小的一个：

```text
结果: 1 -> 1 -> 2 -> 3 -> 4 -> 4
```

为什么最后可以直接接剩余链表？

因为两个原链表本来就是升序的，剩下的部分内部已经有序，直接挂到 `tail.next` 后面即可。

### 易错点

- 最后返回 `dummy.next`，不能返回 `dummy`。
- 每接上一个节点后，要执行 `tail = tail.next`。
- 循环结束后，要把未遍历完的链表接到 `tail.next`。
- 这份写法复用原链表节点，没有创建新节点。

### 复杂度

- 时间复杂度：`O(m + n)`
- 空间复杂度：`O(1)`

## H_234 - 回文链表

### 核心思路

使用快慢指针找到链表中点，然后反转后半段，再从头开始比较：

- `slow` 每次走 1 步，`fast` 每次走 2 步。
- 当前循环结束后，`slow` 指向前半段的最后一个节点。
- 使用 `reverse(slow.next)`，只反转右半段。
- 偶数长度时比较左右两半；奇数长度时自动跳过中间节点。
- `left` 从头节点开始，与反转后的右半段逐个比较节点值。

### 题解代码

```java
public boolean isPalindrome(ListNode head) {
    if (head == null || head.next == null) {
        return true;
    }

    ListNode slow = head;
    ListNode fast = head;

    while (fast.next != null && fast.next.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }

    ListNode reverse = reverse(slow.next);

    ListNode left = head;

    while (reverse != null) {
        if (left.val != reverse.val) {
            return false;
        }
        left = left.next;
        reverse = reverse.next;
    }

    return true;
}

public ListNode reverse(ListNode head) {
    ListNode cur = head;
    ListNode pre = null;

    while (cur != null) {
        ListNode tmp = cur.next;
        cur.next = pre;
        pre = cur;
        cur = tmp;
    }

    return pre;
}
```

### 快速理解

偶数长度：

```text
1 -> 2 -> 2 -> 1
     ^
   slow
```

`reverse(slow.next)` 反转右半段：

```text
2 -> 1
```

得到：

```text
1 -> 2
```

然后和原链表前半部分比较。

奇数长度：

```text
1 -> 2 -> 3 -> 2 -> 1
     ^
   slow
```

`slow.next` 从中间节点后的 `2` 开始：

```text
2 -> 1
```

反转后：

```text
1 -> 2
```

比较过程是：

```text
1 == 1
2 == 2
```

中间节点 `3` 不参与比较，也不影响回文判断。

### 易错点

- 比较节点值用 `left.val != reverse.val`，不是比较节点引用。
- 当前写法会改变原链表结构，因为反转了后半段。
- `while (fast.next != null && fast.next.next != null)` 结束后，`slow` 是前半段末尾，因此应写 `reverse(slow.next)`。
- `head == null` 时不能直接访问 `fast.next`，所以模板开头保留空链表判断。
- 比较循环用 `while (reverse != null)`，只比较右半段长度。

### 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## H_160 - 相交链表

### 核心思路

使用两个指针分别遍历两个链表：

- `pA` 从链表 A 出发。
- `pB` 从链表 B 出发。
- `pA` 走到空后切换到 `headB`。
- `pB` 走到空后切换到 `headA`。
- 如果两个链表相交，两个指针最终会在第一个公共节点相遇。
- 如果不相交，两个指针最终会同时变成 `null`。

### 题解代码

```java
public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
    if (headA == null || headB == null) {
        return null;
    }

    ListNode pA = headA;
    ListNode pB = headB;

    while (pA != pB) {
        if (pA == null) {
            pA = headB;
        } else {
            pA = pA.next;
        }

        if (pB == null) {
            pB = headA;
        } else {
            pB = pB.next;
        }
    }

    return pA;
}
```

### 快速理解

假设：

```text
A 独有长度 = a
B 独有长度 = b
公共长度 = c
```

两个指针走过的路径：

```text
pA: a + c + b
pB: b + c + a
```

总路程一样，所以长度差会被抵消，最终在交点同步相遇。

如果不相交：

```text
pA 和 pB 都会走完 A + B，最后同时到 null
```

### 易错点

- 判断相交用 `pA != pB`，比较的是节点引用，不是节点值。
- 不需要计算两个链表长度。
- 返回 `pA` 即可，相交时是交点，不相交时是 `null`。

### 复杂度

- 时间复杂度：`O(m + n)`
- 空间复杂度：`O(1)`

## H_206 - 反转链表

### 核心思路

把每个节点的 `next` 指针反向：

- `cur` 指向当前正在处理的节点。
- `pre` 指向已经反转好的前一个节点。
- `tmp` 保存原来的下一个节点，避免断链。
- 每轮执行 `cur.next = pre`，然后整体后移。

### 题解代码：迭代

```java
public ListNode reverseList(ListNode head) {
    ListNode cur = head;
    ListNode pre = null;

    while (cur != null) {
        ListNode tmp = cur.next;
        cur.next = pre;
        pre = cur;
        cur = tmp;
    }

    return pre;
}
```

### 题解代码：递归

```java
public ListNode reverseList2(ListNode head) {
    return dfs(head, null);
}

public ListNode dfs(ListNode cur, ListNode pre) {
    if (cur == null) {
        return pre;
    }
    ListNode tmp = cur.next;
    cur.next = pre;
    return dfs(tmp, cur);
}
```

### 快速理解

原链表：

```text
1 -> 2 -> 3 -> null
```

处理节点 `1`：

```text
null <- 1    2 -> 3 -> null
       pre   cur
```

处理节点 `2`：

```text
null <- 1 <- 2    3 -> null
            pre   cur
```

最后返回 `pre`，也就是新头节点。

### 易错点

- 改 `cur.next` 前必须先保存 `tmp = cur.next`。
- 循环结束后 `cur == null`，所以返回的是 `pre`。
- 递归写法空间复杂度是 `O(n)`，因为有递归调用栈。

### 复杂度

- 迭代：时间复杂度 `O(n)`，空间复杂度 `O(1)`
- 递归：时间复杂度 `O(n)`，空间复杂度 `O(n)`

## H_146 - LRU 缓存

### 核心思路

使用 `HashMap + 双向链表` 同时满足按 key 查找和维护最近使用顺序：

- `map`：`key -> Node`，通过 key 在 `O(1)` 时间找到缓存节点。
- 双向链表：在 `O(1)` 时间删除节点、把节点移到头部、淘汰尾部节点。
- `head` 和 `tail` 是虚拟节点；`head.next` 是最近使用，`tail.prev` 是最久未使用。

链表顺序：

```text
head <-> 最近使用 ... 最久未使用 <-> tail
```

### 关键辅助方法

```java
public void remove(Node node) {
    node.prev.next = node.next;
    node.next.prev = node.prev;
}

public void addToHead(Node node) {
    node.next = head.next;
    node.prev = head;
    head.next.prev = node;
    head.next = node;
}
```

- `remove(node)`：把节点从双向链表当前位置摘除。
- `addToHead(node)`：把节点插到头部，表示刚刚被访问或更新。

### 题解代码

```java
public int get(int key) {
    if (!map.containsKey(key)) {
        return -1;
    }

    Node node = map.get(key);
    remove(node);
    addToHead(node);
    return node.v;
}

public void put(int key, int value) {
    if (map.containsKey(key)) {
        Node node = map.get(key);
        node.v = value;
        remove(node);
        addToHead(node);
    } else {
        if (map.size() == capacity) {
            Node prev = tail.prev;
            remove(prev);
            map.remove(prev.k);
        }

        Node node = new Node(key, value);
        addToHead(node);
        map.put(key, node);
    }
}
```

### 快速理解

容量为 `2`：

```text
put(1, 1): head <-> 1 <-> tail
put(2, 2): head <-> 2 <-> 1 <-> tail
get(1):    head <-> 1 <-> 2 <-> tail
put(3, 3): 淘汰 tail.prev，也就是 2
            head <-> 3 <-> 1 <-> tail
```

任何一次 `get`、更新已有 key，都会执行：

```text
remove(node) -> addToHead(node)
```

因此被访问过的节点总是在头部，最久未使用节点总在 `tail.prev`。

### 易错点

- 淘汰的是 `tail.prev`，不是 `tail`；`tail` 是虚拟节点。
- 淘汰节点时，要同时从链表和 `map` 删除。
- 更新已有 key 时，不要创建新节点，更新 `node.v` 后移动到头部即可。
- `capacity == 0` 时，这份代码会错误删除虚拟头节点；若需支持该边界，应在 `put` 开头直接 `return`。

### 复杂度

- `get`：时间复杂度 `O(1)`
- `put`：时间复杂度 `O(1)`
- 空间复杂度：`O(capacity)`

## H_142 - 环形链表 II

### 核心思路

先用快慢指针判断是否有环并找到环内相遇点；再让一个指针从头节点出发，另一个从相遇点出发，二者每次都走 1 步，最终会在环入口相遇。

### 题解代码

```java
public ListNode detectCycle(ListNode head) {
    ListNode slow = head;
    ListNode fast = head;
    ListNode start = head;
    ListNode end = null;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;

        if (slow == fast) {
            end = slow;
            break;
        }
    }

    while (end != null && start != end) {
        start = start.next;
        end = end.next;
    }

    return end;
}
```

### 快速理解

示例：

```text
3 -> 2 -> 0 -> -4
     ^         |
     |_________|
```

第一阶段快慢指针会在环内的 `-4` 相遇：

```text
slow = -4
fast = -4
end  = -4
```

第二阶段：

```text
start 从 3 出发
end   从 -4 出发
```

两者同步走一步后：

```text
start = 2
end   = 2
```

节点 `2` 就是环入口。

### 为什么能找到入口

设：

```text
a：头节点到环入口的距离
b：环入口到相遇点的距离
c：相遇点回到环入口的距离
环长度 L = b + c
```

快慢指针相遇时，慢指针走了 `a + b`；快指针走了慢指针的两倍，并比慢指针多绕了若干圈：

```text
2(a + b) = a + b + kL
```

因此：

```text
a = kL - b = (k - 1)L + c
```

从相遇点走 `a` 步，会先绕若干整圈，再走 `c` 步到入口；从头节点走 `a` 步也正好到入口。因此 `start` 和 `end` 同速前进时会在入口相遇。

### 为什么返回 `end`

第二个循环结束时有两种情况：

- 有环：`start == end`，两者都指向入口，返回任意一个都可以。
- 无环：`end == null`，循环不会执行；此时只有 `return end` 能正确返回 `null`。`start` 仍然指向头节点，不能返回它。

### 易错点

- 第一阶段比较 `slow == fast`，判断的是节点引用，不是节点值。
- `fast != null && fast.next != null` 用于避免 `fast.next.next` 空指针。
- `end` 表示相遇点，不是链表尾节点，命名为 `meet` 会更直观。

### 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## H_19 - 删除链表的倒数第 N 个节点

### 核心思路

使用 `dummy + 快慢指针`：让 `fast` 先走 `n + 1` 步，然后 `fast` 和 `slow` 同时前进。`fast` 到达 `null` 时，`slow` 正好停在待删除节点的前一个节点。

### 题解代码

```java
public ListNode removeNthFromEnd(ListNode head, int n) {
    ListNode dummy = new ListNode(-1);
    dummy.next = head;

    ListNode fast = dummy;
    ListNode slow = dummy;

    for (int i = 0; i <= n; i++) {
        fast = fast.next;
    }

    while (fast != null) {
        fast = fast.next;
        slow = slow.next;
    }

    slow.next = slow.next.next;
    return dummy.next;
}
```

### 为什么 `fast` 走 `n + 1` 步

示例：

```text
dummy -> 1 -> 2 -> 3 -> 4 -> 5
n = 2
```

`fast` 先走 3 步：

```text
dummy -> 1 -> 2 -> 3 -> 4 -> 5
                  ^
                fast
^
slow
```

随后两指针同步前进，直到 `fast == null`：

```text
slow 最终停在 3
slow.next 是 4，也就是倒数第 2 个节点
```

因此直接执行：

```java
slow.next = slow.next.next;
```

就会删除节点 `4`。

### 为什么需要 `dummy`

当删除头节点时：

```text
1 -> 2 -> 3
n = 3
```

`slow` 最终停在 `dummy`，仍然可以通过：

```java
slow.next = slow.next.next;
```

统一删除节点 `1`，最后返回 `dummy.next` 即可。

### 易错点

- `fast` 从 `dummy` 出发时要先走 `n + 1` 步，所以循环是 `i <= n`。
- `slow` 必须从 `dummy` 出发，才能统一处理删除头节点的情况。
- 题目保证 `n` 合法时，`slow.next` 一定不为空，删除时不需要额外 `if` 判断。
- 若不保证 `n` 合法，`for` 循环中应先判断 `fast` 是否为 `null`。

### 复杂度

- 时间复杂度：`O(L)`，`L` 为链表长度。
- 空间复杂度：`O(1)`

## H_2 - 两数相加

### 核心思路

链表中的数字按逆序存储，因此可以从头开始逐位相加。使用 `dummy + tail` 构造结果链表，用 `carry` 保存上一位的进位。

### 题解代码

```java
public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(-1);
    ListNode tail = dummy;

    int carry = 0;
    while (l1 != null || l2 != null || carry != 0) {
        int x = l1 != null ? l1.val : 0;
        int y = l2 != null ? l2.val : 0;

        int sum = x + y + carry;
        carry = sum / 10;

        int cur = sum % 10;
        tail.next = new ListNode(cur);
        tail = tail.next;

        if (l1 != null) {
            l1 = l1.next;
        }
        if (l2 != null) {
            l2 = l2.next;
        }
    }

    return dummy.next;
}
```

### 快速理解

```text
l1: 2 -> 4 -> 3    表示 342
l2: 5 -> 6 -> 4    表示 465

结果: 7 -> 0 -> 8  表示 807
```

逐位计算：

| 位数 | `x` | `y` | 相加前 `carry` | `sum` | 当前位 `sum % 10` | 下一轮 `carry` |
| --- | --- | --- | --- | --- | --- | --- |
| 个位 | 2 | 5 | 0 | 7 | 7 | 0 |
| 十位 | 4 | 6 | 0 | 10 | 0 | 1 |
| 百位 | 3 | 4 | 1 | 8 | 8 | 0 |

### 为什么循环条件包含 `carry != 0`

例如：

```text
99 + 1 = 100

l1: 9 -> 9
l2: 1
结果: 0 -> 0 -> 1
```

两个链表走完后，仍可能有最后一个进位 `1`。因此循环必须写成：

```java
while (l1 != null || l2 != null || carry != 0)
```

### 易错点

- `l1` 和 `l2` 长度可能不同；空链表当前位按 `0` 处理。
- `carry = sum / 10`，当前节点值为 `sum % 10`。
- 每轮都要移动 `tail`，否则结果节点会被覆盖。
- 返回 `dummy.next`，跳过虚拟头节点。

### 复杂度

- 时间复杂度：`O(max(m, n))`
- 结果链表空间复杂度：`O(max(m, n))`
- 除结果链表外的辅助空间：`O(1)`
