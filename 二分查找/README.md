## 4. 易错点

- 先固定区间定义：闭区间 `[left, right]` 或左闭右开 `[left, right)`。
- 求中点使用 `left + (right - left) / 2`。
- 边界二分要明确“找第一个 >= target”还是“找最后一个 <= target”。
- 旋转数组题要先判断哪一半有序，再决定收缩方向。

## 5. 每题笔记

### H_74 - 搜索二维矩阵

- 题目定位：在一个行升序、且每行首元素大于上一行末元素的二维矩阵里查找目标值。
- 当前代码思路：把二维矩阵按行展开成一维有序数组，再做标准二分。
- 关键变量：`left/right` 是一维边界；`mid / n` 和 `mid % n` 用来映射二维位置。
- 解题步骤：定义一维搜索区间；计算中点；映射到二维坐标；根据大小关系移动边界。
- 复杂度：时间 `O(log(mn))`，空间 `O(1)`。
- 面试易错点：不是按“每一行二分”做，而是可以直接一维化成一个整体有序数组。
- 可复述话术：矩阵满足整体有序，所以我把它当成一维数组处理，只是在访问时把下标换算回二维坐标。

### H_153 - 寻找旋转排序数组中的最小值

- 题目定位：在升序数组旋转后的结果中找最小值。
- 当前代码思路：比较 `nums[mid]` 和 `nums[right]`，判断最小值落在左半段还是右半段。
- 关键变量：`left/right` 是搜索边界；`mid` 是中点；`nums[right]` 是当前判断基准。
- 解题步骤：如果 `nums[mid] > nums[right]`，说明最小值在右半段；否则在左半段，包含 `mid`。
- 复杂度：时间 `O(log n)`，空间 `O(1)`。
- 面试易错点：`right = mid` 不是 `right = mid - 1`，因为 `mid` 也可能是最小值。
- 可复述话术：旋转后数组被分成两段有序区间，我只需要看中点和右端点的关系，就能判断最小值在哪一边。
