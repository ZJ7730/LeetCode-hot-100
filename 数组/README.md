## 2. 本专题核心思想

- 数组题优先判断能否原地修改输入，允许修改时经常可以把数组本身当作哈希表。
- 题目给出数字范围 `[1, n]` 时，重点考虑下标映射：数字 `x` 对应下标 `x - 1`。
- 矩阵题通常是边界控制或原地变换，先明确遍历顺序，再写循环。
- Top K 题要根据是否需要稳定顺序、是否允许修改数组，选择堆或快速选择。
- 原地写入类题目要分清读指针和写指针，避免覆盖未处理数据。

## 3. 通用解题模板

### 原地标记模板

对应 `H_448.java`：

```java
for (int i = 0; i < nums.length; i++) {
    int index = Math.abs(nums[i]) - 1;
    nums[index] = -Math.abs(nums[index]);
}

List<Integer> result = new ArrayList<>();
for (int i = 0; i < nums.length; i++) {
    if (nums[i] > 0) {
        result.add(i + 1);
    }
}
return result;
```

核心点：用符号表示数字是否出现过，标记时必须取绝对值。

### 原地哈希交换模板

对应 `H_41.java`：

```java
for (int i = 0; i < nums.length; i++) {
    while (nums[i] > 0 && nums[i] <= nums.length && nums[nums[i] - 1] != nums[i]) {
        int index = nums[i] - 1;
        int temp = nums[index];
        nums[index] = nums[i];
        nums[i] = temp;
    }
}
```

核心点：把数字 `x` 尽量放到下标 `x - 1`，再扫描第一个位置不匹配的下标。

### Top K 小顶堆模板

对应 `H_215.java`：

```java
PriorityQueue<Integer> minHeap = new PriorityQueue<>();

for (int num : nums) {
    minHeap.offer(num);
    if (minHeap.size() > k) {
        minHeap.poll();
    }
}
return minHeap.peek();
```

核心点：堆中只保留最大的 `k` 个数；堆顶是这 `k` 个数里最小的，也就是第 `k` 大。

## 5. 每题详解

### H_448 - 找到所有数组中消失的数字

- 题目定位：数组长度为 `n`，数字范围在 `[1, n]`，找出没有出现过的数字。
- 当前代码思路：遍历数组，把出现过的数字 `x` 对应的下标 `x - 1` 位置标记为负数；最后仍为正数的位置就是缺失数字。
- 关键变量：`index = Math.abs(nums[i]) - 1` 是数字映射到的位置；`result` 保存缺失数字。
- 解题步骤：遍历数组做负数标记；再次遍历数组；如果 `nums[i] > 0`，说明 `i + 1` 没出现过。
- 复杂度：时间 `O(n)`，空间 `O(1)` 不计返回结果。
- 面试易错点：计算下标时必须用 `Math.abs(nums[i])`，否则已经被标记成负数的值会导致下标错误。
- 可复述话术：因为数字范围刚好是 `1` 到 `n`，所以可以把数组下标当作哈希表，用负号记录某个数字是否出现过。

### H_215 - 数组中的第 K 个最大元素

- 题目定位：找数组排序后第 `k` 个最大的元素，不要求返回前 `k` 个，也不要求完整排序。
- 当前代码思路：维护大小为 `k` 的小顶堆；遍历每个数字放入堆，如果堆大小超过 `k`，弹出堆顶最小值。
- 关键变量：`minHeap` 保存当前最大的 `k` 个元素；`minHeap.peek()` 是这 `k` 个元素中最小的，也就是第 `k` 大。
- 解题步骤：创建小顶堆；遍历数组入堆；超过 `k` 时出堆；最终返回堆顶。
- 复杂度：时间 `O(n log k)`，空间 `O(k)`。
- 面试易错点：求第 `k` 大要用小顶堆保留最大 `k` 个数；如果用大顶堆，则需要弹出 `k - 1` 次，复杂度和写法不同。
- 可复述话术：我不需要对整个数组排序，只维护当前最大的 k 个元素，小顶堆堆顶就是这 k 个元素中最小的，即第 k 大。

## 6. 面试高频易错点

- `[1, n]` 范围题经常可以做下标映射，数字 `x` 对应位置 `x - 1`。
- 原地负数标记会改变输入，后续读取必须用绝对值。
- 原地交换时要防止死循环，交换条件里需要判断目标位置是否已经是正确数字。
- 矩阵边界模拟要在每次移动边界后判断是否还合法。
- 从后往前合并数组可以避免覆盖还没处理的有效元素。
- Top K 用小顶堆时，堆大小超过 `k` 就弹出堆顶，最终堆顶才是第 `k` 大。
- `215` 只要求第 `k` 大，不要求前 `k` 个元素有序。
