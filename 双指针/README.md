## 2. 本专题核心思想

- 双指针的关键是明确两个指针各自的语义，而不是机械移动。
- 对撞指针通常用于有左右边界的问题，每次根据局部条件缩小搜索范围。
- 快慢指针常用于原地覆盖、删除元素、移动元素。
- 快慢指针还常用于“数组映射成链表”的场景，例如寻找环入口。
- 三指针适合把数组原地分成多个区域，例如 0/1/2 分类。
- 滑动窗口适合连续子串或连续子数组问题，本质是维护一个合法区间 `[left, right]`。
- 两次遍历找边界的问题，关键是从两个方向分别找出"破坏有序"的最远位置。
- 面试中要说清楚为什么移动某个指针不会漏答案。

## 3. 通用解题模板

### 对撞指针模板

对应 `H_11.java`：

```java
int left = 0;
int right = nums.length - 1;
int res = 0;

while (left < right) {
    // 根据 left/right 计算答案
    res = Math.max(res, 当前答案);

    if (应该移动左边界) {
        left++;
    } else {
        right--;
    }
}
return res;
```

### 快慢指针 / 原地覆盖模板

对应 `H_283.java`：

```java
int left = 0;
for (int right = 0; right < nums.length; right++) {
    if (nums[right] 满足保留条件) {
        nums[left] = nums[right];
        left++;
    }
}

for (int i = left; i < nums.length; i++) {
    nums[i] = 默认值;
}
```

`left` 表示下一个写入位置，`right` 负责扫描原数组。

### 快慢指针 / 环检测模板

对应 `H_287.java`：

```java
int slow = 0;
int fast = 0;

do {
    slow = nums[slow];
    fast = nums[nums[fast]];
} while (slow != fast);

int slowStart = 0;
while (slowStart != slow) {
    slowStart = nums[slowStart];
    slow = nums[slow];
}
return slowStart;
```

这个模板的核心是：
- 第一次相遇说明存在环。
- 第二次同步前进时，相遇点就是环入口，也就是重复数字。

### 三指针模板

对应 `H_75.java`：

```java
int low = 0;
int mid = 0;
int high = nums.length - 1;

while (mid <= high) {
    if (nums[mid] == 0) {
        // 交换 low 和 mid，两个指针同时右移
    } else if (nums[mid] == 1) {
        mid++;
    } else {
        // 交换 mid 和 high，high 左移，mid 不动
    }
}
```

核心不变量是：
- `[0, low)` 全是 0
- `[low, mid)` 全是 1
- `(high, n - 1]` 全是 2

### 滑动窗口模板

对应 `H_3.java`：

```java
HashSet<Character> window = new HashSet<>();
int left = 0;
int res = 0;

for (int right = 0; right < s.length(); right++) {
    char rightChar = s.charAt(right);

    while (窗口不合法) {
        window.remove(s.charAt(left));
        left++;
    }

    window.add(rightChar);
    res = Math.max(res, right - left + 1);
}
return res;
```

对应 `H_76.java` 的最小覆盖子串，也是滑动窗口，只是窗口合法性的判断从“没有重复”变成“覆盖目标串 t”。

## 5. 每题详解

### H_3 - 无重复字符的最长子串

- 题目定位：求不含重复字符的最长连续子串长度。
- 当前代码思路：`HashSet<Character> window` 保存当前窗口字符；右指针扩张，遇到重复就移动左指针直到不重复。
- 关键变量：`left` 是窗口左边界；`right` 是窗口右边界；`res` 是最大窗口长度。
- 解题步骤：遍历 `right`；如果 `rightChar` 已在窗口中，就不断移除 `left` 字符；加入 `rightChar`；更新答案。
- 复杂度：时间 `O(n)`，空间 `O(字符集大小)`。
- 面试易错点：收缩窗口要用 `while`，因为可能需要连续移除多个字符。
- 可复述话术：我维护一个没有重复字符的窗口，右边界负责加入新字符，一旦重复就移动左边界直到窗口重新合法。

### H_11 - 盛最多水的容器

- 题目定位：选两条竖线，使它们和 x 轴组成的容器装水最多。
- 当前代码思路：左右指针从两端开始，计算面积后移动高度较小的一边。
- 关键变量：面积是 `(right - left) * Math.min(height[left], height[right])`。
- 解题步骤：初始化两端指针；计算当前面积；更新最大值；移动较短边；直到指针相遇。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：移动较高边没有意义，因为宽度变小且短板不变，面积不可能更大。
- 可复述话术：容器面积受较短边限制，缩小宽度时只有移动短板才可能找到更高的短板，从而获得更大面积。

### H_283 - 移动零

- 题目定位：把数组中的 0 移到末尾，同时保持非零元素相对顺序。
- 当前代码思路：`right` 扫描数组，遇到非零元素写到 `left` 位置，然后 `left++`；最后从 `left` 到末尾补 0。
- 关键变量：`left` 表示下一个非零元素应该写入的位置。
- 解题步骤：第一遍压缩所有非零元素；第二遍把剩余位置填成 0。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：这是稳定移动，不能简单排序或交换导致非零元素相对顺序变化。
- 可复述话术：先把所有非零元素按原顺序覆盖到数组前面，再把后面的位置统一置零。

### H_287 - 寻找重复数

- 题目定位：数组长度为 `n + 1`，元素范围是 `[1, n]`，找出唯一重复的数字。
- 当前代码思路：把下标和数值的关系看成“链表”，`i -> nums[i]`；由于有重复值，这条链表一定会形成环，重复数字就是环入口。
- 关键变量：`slow` 和 `fast` 用来在环内相遇；`slowStart` 从起点出发，用于第二阶段寻找环入口。
- 解题步骤：
  1. 初始化两个指针都从下标 `0` 出发。
  2. 慢指针每次走一步，快指针每次走两步，直到两者相遇。
  3. 让一个指针回到起点，另一个指针留在相遇点。
  4. 两个指针每次都走一步，再次相遇的位置就是环入口，也就是重复数字。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：这里不是在数组里直接找“值重复”，而是在“下标映射”形成的环里找入口。
- 可复述话术：我把数组看成一个函数图，重复数字会让路径进入环；先用快慢指针确认有环，再用同速前进找到环入口。

### H_5 - 最长回文子串

- 题目定位：在字符串中找到最长的回文连续子串。
- 当前代码思路：`dp[i][j]` 表示 `s[i..j]` 是否回文；如果两端字符相同，并且内部区间也是回文，那么当前区间就是回文。
- 关键变量：`start` 记录最长回文的起点；`len` 记录最长回文长度。
- 解题步骤：从后往前枚举左端点 `i`，从 `i` 往后枚举右端点 `j`；判断区间是否回文；如果是就更新答案。
- 复杂度：时间 `O(n^2)`，空间 `O(n^2)`。
- 面试易错点：`substring(start, end)` 的第二个参数是结束下标，不是长度。
- 可复述话术：我用二维 DP 判断每个区间是否是回文，只要一个区间合法，就用它更新当前最长答案。

### H_75 - 颜色分类

- 题目定位：把只包含 0、1、2 的数组原地排序。
- 当前代码思路：维护三个区域，`low` 前面全是 0，`mid` 扫描未知区域，`high` 后面全是 2。
- 关键变量：`low`、`mid`、`high` 分别表示 0 区、扫描指针和 2 区边界。
- 解题步骤：当 `nums[mid]` 为 0 时交换到前面；为 1 时直接右移；为 2 时交换到后面，`mid` 不动继续检查新值。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：交换到 `mid` 的新元素还没检查，所以遇到 2 时不能直接移动 `mid`。
- 可复述话术：我把数组分成 0 区、未知区和 2 区，扫描指针只负责处理未知区，直到整个数组被分完。

### H_76 - 最小覆盖子串

- 题目定位：在 `s` 中找到包含 `t` 所有字符的最短子串。
- 当前代码思路：滑动窗口先右扩张，直到窗口覆盖 `t`；再左收缩，尽可能缩短长度。
- 关键变量：`need` 记录目标串需要的字符次数；`window` 记录当前窗口字符次数；`formed` 记录已经满足需求的字符种类数。
- 解题步骤：
  1. 统计 `t` 中每个字符的需求次数。
  2. 右指针扩张窗口，更新窗口计数。
  3. 当窗口已经覆盖 `t` 时，尝试移动左指针缩小窗口。
  4. 每次窗口合法时更新最短答案。
- 复杂度：时间 `O(n)`，空间 `O(字符集大小)`。
- 面试易错点：窗口合法后要继续收缩，直到不合法为止，这样才能保证最短。
- 可复述话术：我先把窗口扩到能覆盖目标串，再不断缩小左边界找最短合法窗口，最后保留全局最优。

### H_581_ - 最短无序连续子数组

- 题目定位：找到最短的一个连续子数组，只要将它排序，整个数组就变成升序。
- 当前代码思路：两次遍历，分别从左往右找右边界、从右往左找左边界。如果数组完全有序，从左往右每个元素都 >= 前面最大值，从右往左每个元素都 <= 后面最小值；违反规则的位置就是无序子数组的边界。
- 关键变量：`max` 记录从左往右遍历过的最大值；`min` 记录从右往左遍历过的最小值；`right` 记录无序子数组的右边界；`left` 记录无序子数组的左边界。
- 解题步骤：
  1. 从左往右遍历，维护 `max`；若 `nums[i] < max`，说明 `i` 破坏了升序，更新 `right = i`；否则更新 `max`。
  2. 从右往左遍历，维护 `min`；若 `nums[i] > min`，说明 `i` 破坏了降序（从右往左看应递减），更新 `left = i`；否则更新 `min`。
  3. 若 `right == -1` 说明数组已有序返回 0；否则返回 `right - left + 1`。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：两个方向的遍历不能只做一个，仅从左往右找不到左边界，仅从右往左找不到右边界。
- 可复述话术：我从两个方向各遍历一次，从左往右找最右的破坏升序位置作为右边界，从右往左找最左的破坏降序位置作为左边界，两者之间就是要排序的子数组。

## 6. 面试高频易错点

- 对撞指针要能证明移动哪一边不会漏掉最优解。
- 滑动窗口更新答案的位置要放在窗口合法之后。
- 窗口收缩经常需要 `while`，不是 `if`。
- 快慢指针原地覆盖时，`left` 通常表示写入位置，而不是遍历位置。
- 原地修改数组要注意是否要求保持相对顺序。
