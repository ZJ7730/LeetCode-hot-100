# 双指针专题面试复习

## 1. 题目总览

| 文件 | 题号 | 题目 | 核心方法 | 模板类型 | 复杂度 |
| --- | --- | --- | --- | --- | --- |
| `H_3.java` | 3 | 无重复字符的最长子串 | `HashSet` 维护窗口，重复时移动左边界 | 滑动窗口 | 时间 `O(n)`，空间 `O(字符集大小)` |
| `H_11.java` | 11 | 盛最多水的容器 | 左右指针对撞，每次移动较短边 | 对撞指针 | 时间 `O(n)`，空间 `O(1)` |
| `H_283.java` | 283 | 移动零 | `right` 扫描非零元素，`left` 表示写入位置 | 快慢指针 / 原地覆盖 | 时间 `O(n)`，空间 `O(1)` |

## 2. 本专题核心思想

- 双指针的关键是明确两个指针各自的语义，而不是机械移动。
- 对撞指针通常用于有左右边界的问题，每次根据局部条件缩小搜索范围。
- 快慢指针常用于原地覆盖、删除元素、移动元素。
- 滑动窗口适合连续子串或连续子数组问题，本质是维护一个合法区间 `[left, right]`。
- 面试中要说清楚为什么移动某个指针不会漏答案。

## 3. 通用解题模板

### 对撞指针模板

对应 `H_11.java`：

```java
int left = 0;
int right = nums.length - 1;
int res = 0;

while (left < right) {
    // 根据 left/right 计算答案
    res = Math.max(res, 当前答案);

    if (应该移动左边界) {
        left++;
    } else {
        right--;
    }
}
return res;
```

### 快慢指针 / 原地覆盖模板

对应 `H_283.java`：

```java
int left = 0;
for (int right = 0; right < nums.length; right++) {
    if (nums[right] 满足保留条件) {
        nums[left] = nums[right];
        left++;
    }
}

for (int i = left; i < nums.length; i++) {
    nums[i] = 默认值;
}
```

`left` 表示下一个写入位置，`right` 负责扫描原数组。

### 滑动窗口模板

对应 `H_3.java`：

```java
HashSet<Character> window = new HashSet<>();
int left = 0;
int res = 0;

for (int right = 0; right < s.length(); right++) {
    char rightChar = s.charAt(right);

    while (窗口不合法) {
        window.remove(s.charAt(left));
        left++;
    }

    window.add(rightChar);
    res = Math.max(res, right - left + 1);
}
return res;
```

## 4. 题目分类

- 滑动窗口：`H_3.java`
- 对撞指针：`H_11.java`
- 快慢指针 / 原地覆盖：`H_283.java`

## 5. 每题详解

### H_3 - 无重复字符的最长子串

- 题目定位：求不含重复字符的最长连续子串长度。
- 当前代码思路：`HashSet<Character> window` 保存当前窗口字符；右指针扩张，遇到重复就移动左指针直到不重复。
- 关键变量：`left` 是窗口左边界；`right` 是窗口右边界；`res` 是最大窗口长度。
- 解题步骤：遍历 `right`；如果 `rightChar` 已在窗口中，就不断移除 `left` 字符；加入 `rightChar`；更新答案。
- 复杂度：时间 `O(n)`，空间 `O(字符集大小)`。
- 面试易错点：收缩窗口要用 `while`，因为可能需要连续移除多个字符。
- 可复述话术：我维护一个没有重复字符的窗口，右边界负责加入新字符，一旦重复就移动左边界直到窗口重新合法。

### H_11 - 盛最多水的容器

- 题目定位：选两条竖线，使它们和 x 轴组成的容器装水最多。
- 当前代码思路：左右指针从两端开始，计算面积后移动高度较小的一边。
- 关键变量：面积是 `(right - left) * Math.min(height[left], height[right])`。
- 解题步骤：初始化两端指针；计算当前面积；更新最大值；移动较短边；直到指针相遇。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：移动较高边没有意义，因为宽度变小且短板不变，面积不可能更大。
- 可复述话术：容器面积受较短边限制，缩小宽度时只有移动短板才可能找到更高的短板，从而获得更大面积。

### H_283 - 移动零

- 题目定位：把数组中的 0 移到末尾，同时保持非零元素相对顺序。
- 当前代码思路：`right` 扫描数组，遇到非零元素写到 `left` 位置，然后 `left++`；最后从 `left` 到末尾补 0。
- 关键变量：`left` 表示下一个非零元素应该写入的位置。
- 解题步骤：第一遍压缩所有非零元素；第二遍把剩余位置填成 0。
- 复杂度：时间 `O(n)`，空间 `O(1)`。
- 面试易错点：这是稳定移动，不能简单排序或交换导致非零元素相对顺序变化。
- 可复述话术：先把所有非零元素按原顺序覆盖到数组前面，再把后面的位置统一置零。

## 6. 面试高频易错点

- 对撞指针要能证明移动哪一边不会漏掉最优解。
- 滑动窗口更新答案的位置要放在窗口合法之后。
- 窗口收缩经常需要 `while`，不是 `if`。
- 快慢指针原地覆盖时，`left` 通常表示写入位置，而不是遍历位置。
- 原地修改数组要注意是否要求保持相对顺序。

## 7. 推荐刷题顺序

1. `H_283.java`：先掌握快慢指针和原地覆盖。
2. `H_11.java`：练习对撞指针和移动策略证明。
3. `H_3.java`：掌握滑动窗口的扩张、收缩和更新答案。
